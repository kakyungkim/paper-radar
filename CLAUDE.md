# paper-radar

매주 신뢰할 수 있는 프리프린트·저널을 스캔해 **방법·임상·산업** 3렌즈로 정리하고, 그 누적을 **동향 리포트 → 스터디 발제 → 블로그/리뷰**로 자라게 하는 개인 지식 자산 하네스다. **바이오인포매틱스·유전체·임상 ML + 신약개발 AI**에 포커스한다. (econ-radar의 자매 하네스 — 검증된 1층/2층/3층 구조를 논문 도메인에 이식했다.)

## 사용자의 전략적 북극성 (모든 에이전트가 반영)
- 정체성: **임상 데이터·유전체·머신러닝을 넘나드는 바이오인포매틱스 연구자.** 장기 목표는 **신약개발 총괄(R&D 리더)** 시점 기르기.
- 읽는 목적: 최신 논문을 단순 요약이 아니라 **"이 방법을 내 연구에 쓸까", "임상에서 통할까", "신약개발·산업에 무슨 함의인가"** 세 각도로 해석.
- 발제: 팀 스터디에서 바로 쓸 수 있는 **발제 자료**를 매주 한 편 뽑는다. 학계 관점과 산업 관점을 나눠 보는 스터디 전통을 잇는다.
- 소스: 프리프린트(bioRxiv·medRxiv·arXiv) 우선 + PubMed·주요 저널. **영어 논문은 한국어로 번역·요약**하되 전문용어는 한국어(영문) 병기.

## 포커스 범위 (paper-scout 필터 기준) — "핵심 + 와이드" 2단
전 분야를 보되 핵심 축에 비중을 둔다(econ-radar의 "전 분야 다루되 제약/바이오/AI 비중"과 동일 패턴). 시야가 좁아지지 않게 매주 와이드 단을 둔다.
- **핵심 축 (매주 깊게, 5편 안팎 — vault 누적·발제 대상)**: 바이오인포매틱스(유전체·전사체·멀티오믹스), 임상 ML/예측모델, 신약개발 AI(타깃 발굴·분자 생성·단백질 구조·임상시험 최적화). 위에 직접 닿는 LLM/파운데이션 모델 방법론(생물학 적용에 한해) 포함.
- **🔭 와이드 (매주 2~3편, 가볍게 — 한눈에 요약만, 누적은 선택)**: 그 주 화제가 된 다른 분야 논문. 기본값은 **그 주 화제작 자유 선택**(일반 AI 에이전트·다른 생물학 분야·흥미로운 방법론 등). 단골 시드 분야가 정해지면 여기 명시: _(현재 시드 없음 — 자유 선택)_.
- **승격 규칙**: 와이드에서 특정 주제가 반복해서 끌리면 정식 핵심 축/렌즈로 승격 검토(improvement-log에 기록).
- **제외**: 핵심·와이드 어디에도 방법·임상·산업 함의가 없는 순수 결과 나열 논문, 도메인 무관 ML 일반론.

## 구조 (3층)
- **1층(매주)**: paper-scout → method-analyst ∥ clinical-analyst ∥ industry-analyst → digest-editor → **style-critic(문체)** → **claim-checker(수치·출처 검증)** → digest-renderer(`scripts/render_html.py`) → knowledge-curator
- **2층(자산)**: `vault/` = 옵시디언 vault. 논문 노트·태그·`[[링크]]`·주제 MOC로 누적.
- **3층(저술)**: study-brief-writer(스터디 발제) · trend-synthesizer(동향) → (선택) 블로그/리뷰.
- **품질 기준**: `vault/_meta/benchmarks.md`(본받을 논문 다이제스트·리뷰 매체)를 깊이·문체 기준선으로.

## 3렌즈 — 논문을 보는 세 각도
1. **방법 (Method)** — 모델·데이터·신규성. "무엇이 새로운가, 재현 가능한가, 내 연구에 쓸 수 있나." 코드/데이터 공개 여부, 베이스라인, 한계.
2. **임상 (Clinical)** — 임상적 의미·검증 수준. "환자·진단·치료에 어떤 영향인가, 검증은 어디까지(in silico/전향·후향/외부검증)인가." 과대해석 경계.
3. **산업 (Industry)** — 신약개발·상용화·경쟁 구도 함의. "어떤 회사·파이프라인·모달리티에 닿나, R&D 의사결정 관점에서 무슨 신호인가." (econ-radar 유망기업 렌즈와 교차.)

## 문체·깊이 원칙 (모든 산출물)
- 사람이 쓴 것처럼 자연스럽게. AI 상투구("A가 아니라 B", "~인 셈") 금지. 전문용어는 한국어(영문) 병기.
- 2층 구조: 한눈에(TL;DR) + 더 깊이 보기(심층). 모든 논문에 **DOI/원문 링크**와 **1차 자료(코드·데이터셋)** 링크.
- 깊이 장치: Why it matters 한 줄 · 방법의 핵심 가정 · 검증 수준 명시 · 양면 시각(강점/한계) · "내 연구·R&D에 어떻게" 전략 노트(절제된 표현).

## 안전선
- 외부 발송(메일·메신저 푸시, 블로그/리뷰 게시)은 **사람 승인 뒤**. (텔레그램 채널 알림은 econ-radar처럼 자동화 검토 — 단, 발제·리뷰 발행은 사람 판단.)
- **논문 claim을 사실로 단정하지 않는다.** 검증 수준(in silico/전향·후향/외부검증 유무)을 항상 명시. 출처(DOI) 없는 수치 생성 금지. 프리프린트는 "미동료심사(preprint)"를 표기.
- 저작권: 논문 본문 대량 복제 금지. 요약·인용은 출처 명시로.

## econ-radar에서 가져온 것 (재사용 자산)
- 1층/2층/3층 골격, 이중 게이트(문체+사실), 결정적 스크립트 렌더(`render_html.py` + 양식 계약), 주간 클라우드 루틴 패턴, run-metrics 계측, improvement-log 운영 규율.
- 차이: 렌즈가 산업구조·투자·시장 → **방법·임상·산업**. 소스가 경제지 → **프리프린트·저널**. dedup이 헤드라인 → **DOI/arXiv ID 기준**. 발행물에 **스터디 발제 자료**가 추가됨.
