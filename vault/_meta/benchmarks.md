# paper-radar 벤치마크

> 목적: 논문 요약을 "초록 다시 쓰기"가 아니라 **읽는 이가 현장에서 쓸 수 있는 해석**으로 끌어올린다.
> 조사 방식: WebSearch/WebFetch로 실제 매체 확인(2026-06-15). 접근 실패 항목은 명시.
> 관련 문서: [[improvement-log]], [[lens-guide]], `CLAUDE.md`(3렌즈: 방법·임상·산업)

---

## 바로 반영할 체크리스트

digest-editor·style-critic이 매 논문 항목에서 직접 대조한다.

- [ ] **Why it matters 필수** — "이 논문이 왜 중요한가"를 1~2문장으로. 단순 "흥미롭다" 금지. 임상/산업/방법론 중 어느 축에 닿는지 명시.
- [ ] **방법의 핵심 1줄** — 무엇이 기존과 다른가(신규성 핵심 포인트). 모델명·데이터셋·평가지표를 구체적으로.
- [ ] **검증 수준 명시** — peer-reviewed / preprint(미동료심사) / conference 구분. 외부 검증 여부("독립 코호트로 검증" 또는 "in silico 한정").
- [ ] **강점/한계 양면** — 저자가 명시한 한계 or 다음 단계. "한계 없이 완벽한 논문"은 없다.
- [ ] **1차 자료 링크** — DOI + 코드/데이터셋 공개 여부(GitHub·Zenodo·HuggingFace 등). 없으면 "코드 미공개" 표기.
- [ ] **수요 렌즈 1줄** — 누가 이 기술을 사고 싶어할까. 환자·처방의·제약사·투자자 중 주된 수요자.
- [ ] **논문당 분량 상한** — 핵심 항목 250~350단어, 와이드 항목 100~150단어. 길이로 깊이를 흉내 내지 않기.

## 참고 매체

### A. 국내 — BRIC 한빛사

- **URL**: https://www.ibric.org/bric/hanbitsa/han-interview.do
- **강점**: 저자 1인칭 서술. 연구 동기("기존 연구들이 심근 자체에만 초점을 맞춰 온 점에 문제의식을 느끼고…")가 들어가 학술 논리의 속살이 드러난다. 전문용어를 쓰되 맥락으로 설명. 소속·ORCID까지 기재해 신뢰도 확보.
- **구조**: 논문 분야 소개 → 연구 배경·문제의식 → 핵심 발견 → 의의 및 전망 → (선택) 후배 조언. 1인칭 담백체.
- **paper-radar에 적용할 요소**: "문제의식 한 줄" 장치. 발견을 나열하기 전에 **왜 이 질문이 필요했는지** 한 문장을 선행시킨다. 국내 저자 논문의 경우 한빛사 인터뷰 링크를 부기.
- **표본 발췌**: "비만 조건에서 심장지방조직 내 지질 대사를 담당하는 페록시좀의 기능이 현저히 저하된다는 점을 규명했습니다. 조직 간 상호작용이라는 새로운 관점에서 접근한 연구입니다."
- **접근 결과**: 실제 페이지 확인 성공.

### B. 국내 — 사이언스타임즈

- **URL**: https://www.sciencetimes.co.kr/nscvrg/list/menu/251 (생명과학·의학 섹션)
- **강점**: 과학 저널리즘 3단 구조가 명확하다 — ① 핵심 성과 리드(1문장) → ② 문제 제시(기존 한계) → ③ 해결 방법 → ④ 성과·응용 → ⑤ 향후 의의. 기술용어(FRET, BRET)를 풀어쓰기와 병기하는 방식이 독자 친화적이다.
- **구조**: 제목에 동사 포함("~를 개발했다", "~를 밝혔다"). 리드문에 연구팀 소속·성과 요지 포함.
- **paper-radar에 적용할 요소**: ① 제목을 결과 동사로 끝내기 — "AlphaFold3 적용, 항체-항원 결합력 예측 정확도 향상". ② "기존 한계 → 이번 해결책" 두 줄 페어. ③ 게재 저널명과 날짜를 본문 말미에 한 줄로.
- **표본 발췌**: "국내 연구진이 암 발생의 주요 원인으로 꼽히는 KRAS 유전자의 활성화 과정을 살아있는 세포에서 실시간으로 관찰할 수 있는 차세대 바이오센서 기술을 개발했다."
- **접근 결과**: 실제 페이지 확인 성공.

---

### C. 해외 — Import AI (Jack Clark)

- **URL**: https://jack-clark.net/ · https://importai.substack.com/
- **강점**: 논문마다 **"Why this matters"** 섹션이 독립 단락으로 고정된다. 단순 "흥미롭다" 수준이 아니라 "AI가 이렇게 인간의 건강에 기여하는 방식이다" 같은 거시 함의까지 끌어올린다. 반대로 리스크(바이오무기·편향)도 솔직하게 붙인다.
- **구조**: 굵은 제목·이탤릭 부제 → What it is → What they did(방법·수치) → Why this matters → Read more(링크). 항목당 약 200~300단어.
- **paper-radar에 적용할 요소**: ① "Why this matters" 단락을 고정 장치로. ② 수치를 인용구(*"72개 샌드박스 환경"*)로 강조. ③ 임상·산업 함의가 없는 ML 논문도 "논문이 현실에서 쓰인다면" 시나리오 1줄.
- **표본 발췌**: "Why this matters – this is how AI delivers benefits to the world: Tools like the ESM family of technologies are how human scientists are going to team up with AI systems to improve human health around the world."
- **접근 결과**: 실제 페이지 확인 성공. 일부 이슈는 단일 장문 에세이 형태.

### D. 해외 — Asimov Press

- **URL**: https://press.asimov.com/
- **강점**: "과학은 마법이 아니다 — 모든 성과에는 물리적 설명이 있다"는 편집 철학. **메커니즘 중심 서술** — 과정을 생략하지 않고 "왜 이 방법이 작동하는가"를 끝까지 따라간다. Long Reads는 2,000~8,000단어로 깊이 자체가 차별점.
- **구조**: 서사적 도입(문제의 역사·배경) → 핵심 메커니즘 해설 → 현재 임상/산업 상황 → 전망과 남은 과제. 경제·사회 함의까지 확장.
- **paper-radar에 적용할 요소**: ① 월간 동향·스터디 발제에서 "메커니즘 깊이 박스" — 방법의 핵심 가정을 2~3줄로 설명. ② "이 분야의 역사적 맥락 한 줄" — 왜 이 문제가 오래 풀리지 않았나. ③ Asimov 방식의 경제·규제 연결은 산업 렌즈에 흡수.
- **표본 발췌**: "How to Build a Strep A Vaccine" — 과거 지연 요인(자금·타깃 논란) → 현재 임상시험 진행 → 향후 필요한 것(투자·정책)의 흐름. 과학 내용만이 아니라 "왜 아직 없는가"를 정면으로 다룸.
- **접근 결과**: 실제 페이지 확인 성공. 일부 글은 구독 필요.

---

### E. 해외 — The Batch (DeepLearning.AI)

- **URL**: https://www.deeplearning.ai/the-batch/
- **강점**: AI 연구를 "초보자도 이해할 수 있는 언어"로 번역하면서도 전문가가 읽어도 정밀하다. Andrew Ng의 주간 서신이 독자와의 목소리 연결점. 각 항목이 독립 단위로 짧고 완결된다.
- **구조**: 카테고리(Research·Business·Culture·Hardware) → 짧은 항목 묶음. 항목당 100~200단어, 전문용어는 인라인 설명.
- **paper-radar에 적용할 요소**: ① 와이드 단의 논문은 The Batch 스타일 — 100~150단어 내 완결. ② 분야 배지(Method / Clinical / Industry)로 카테고리 명시. ③ "누구를 위한 연구인가"를 첫 문장에.
- **표본 발췌**: (직접 페이지 접근 부분적 성공. 이슈 아카이브 목록 확인 — 실제 본문은 개별 링크 진입 필요)
- **접근 결과**: 아카이브 목록 확인. 본문 형식은 검색 결과 기반 분석.

---

### F. 해외 — Owl Posting (Abhishaike Mahajan)

- **URL**: https://www.owlposting.com/
- **강점**: 생물학×ML 교차 영역을 **심층 에세이**로 다룬다. 특정 과학 개념(TIGIT, perturbation 등)을 주제로 잡아 2,800~8,400단어로 파고든다. 일반 논문 요약이 아니라 "이 개념이 왜 중요한 연구 흐름인가"를 주제 중심으로 재편.
- **구조**: 단일 주제 심층 에세이. 월 2~3회 발행. 문체는 기술적이면서도 접근 가능.
- **paper-radar에 적용할 요소**: ① 3층(저술)의 월간 동향 리뷰에서 단일 주제를 이 방식으로. ② 주간 다이제스트에서는 "이 논문이 더 큰 흐름(예: 단백질 구조 예측의 패러다임 전환)의 어디쯤인가" 1줄 맥락 배치.
- **표본 발췌**: 암 백신·바이오테크 금융·TIGIT 등 — 한 편이 12~38분 읽기 분량의 깊이.
- **접근 결과**: 실제 페이지 확인 성공.

### G. 해외 — HuggingFace Daily Papers

- **URL**: https://huggingface.co/papers
- **강점**: 커뮤니티가 매일 arXiv에서 가장 주목받는 논문을 선별. **집단지성 필터링** — 좋아요·댓글 수가 해당 주의 "화제 논문" 신호. 소속 기관 표기로 출처 신뢰도 빠르게 파악.
- **구조**: 목록 페이지는 제목·기관·상호작용 지표. 개별 페이지에서 초록·커뮤니티 논의 확인.
- **paper-radar에 적용할 요소**: ① paper-scout의 소스 시드로 직접 사용. ② 좋아요 수를 "화제성" 신호로 병기(예: "HF Daily Papers #3, 좋아요 142"). ③ 커뮤니티 댓글에서 "실무자 반응"을 수요 렌즈 힌트로 활용.
- **표본 발췌**: 목록 예시 — "EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments" (MIT), "MiniMax Sparse Attention" (MiniMax AI).
- **접근 결과**: 실제 페이지 확인 성공.

## 깊이 장치 (논문 해설 특화)

매 논문 항목에서 해당하는 장치를 선택적으로 적용한다. 전부 쓸 필요 없음 — 논문 유형에 맞는 것 2~3개.

| 장치 | 한 줄 설명 | 적합한 논문 유형 |
|------|-----------|----------------|
| **Why it matters** | 이 연구가 왜 지금 중요한가 (임상/산업/방법론 연결) | 모든 논문 필수 |
| **문제의식 선행** | "왜 이 질문이 필요했나" 1문장 → 발견 서술 (한빛사 방식) | 기초연구·방법론 논문 |
| **기존 한계 → 이번 해결** | 기존 방법의 병목 → 이 논문이 뚫은 방식 (사이언스타임즈 방식) | 비교 연구·벤치마크 |
| **메커니즘 핵심 가정** | 이 방법이 작동하는 이유를 2~3줄로 (Asimov Press 방식) | 알고리즘·모델 논문 |
| **수요 렌즈** | 누가 이 기술을 사고 싶어할까 (환자·처방의·제약사·투자자) | 임상 적용 논문 |
| **검증 수준** | peer-reviewed / preprint / conference + 외부 검증 여부 | 모든 논문 필수 |
| **역사적 맥락** | 왜 이 문제가 오래 풀리지 않았나 1줄 (Asimov 방식) | 오랜 난제 해결 논문 |
| **화제성 신호** | HF Daily Papers 좋아요·순위, Twitter 언급 등 | 와이드 단 논문 |

## 하지 말아야 할 것

조사한 매체들의 안티패턴을 귀납. digest-editor·style-critic이 교차 점검.

- **초록 복붙** — 초록을 한국어로 옮기는 것은 번역이지 해설이 아니다. "왜 중요한가"가 없으면 반려.
- **수치 나열** — "정확도 0.92, AUC 0.87, F1 0.84" 식의 나열. 수치는 **비교 맥락**(기존 SOTA 대비, 외부 코호트에서)과 함께만.
- **장황한 방법론** — 아키텍처 레이어 수·학습률 같은 세부 하이퍼파라미터는 다이제스트에 불필요. 핵심 설계 아이디어만.
- **출처 없는 수치** — DOI·코드·데이터 링크 없는 수치는 생성 금지. 확인 불가 시 "원문 확인 필요" 표기.
- **프리프린트를 사실로 단정** — "~임이 밝혀졌다"가 아니라 "~임을 시사한다(preprint, 동료심사 전)" 표기.
- **임상 과대해석** — in silico 결과를 "치료제로 쓸 수 있다"로 연결하는 비약.
- **긍정 일변도** — "놀라운 성과" 식의 찬사. 한계·다음 단계를 반드시 병기(Import AI 방식: 리스크도 솔직하게).
- **AI 상투구** — "A가 아니라 B", "~인 셈이다", "~해야 한다" 식의 기계적 문체.
