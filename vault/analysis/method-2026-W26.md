## 1. DeepBD: 유전성 출생결함 변이 우선순위 지정 에이전트 워크플로

**arXiv**: https://arxiv.org/abs/2606.24779 (arXiv:2606.24779 · 2026-06-23 · **preprint, 미동료심사**)
**저자**: Shiyu Li, Ziqi Yan 외 | Zhejiang University

**문제의식 선행 — 왜 이 질문이 필요했나**
유전성 출생결함 진단에서 Exomiser 같은 규칙 기반 도구와 LLM 단독 재순위화는 각각 증거 타입 하나씩만 소화하며, 분자 메커니즘 맥락과 전문가 세분화를 동시에 다루지 못했다. 18,622건 규모의 태아·영아 코호트가 있음에도 체계적 에이전트 워크플로가 없었다는 것이 이 논문의 출발점이다.

**무엇이 새로운가**

주장된 신규성: LLM 보조 케이스 구조화 → 사전학습 증거 엔진 → 전문가 모듈 → 근거 기반 진단 검토 레이어의 4단 에이전트 워크플로.

실제로 새로운 점을 구분하면 두 가지다. (1) 증거 엔진의 사전학습 방식: 인과 변이 라벨 없이 자기지도 대조학습(self-supervised contrastive pretraining)으로 증거 인코더를 초기화한 뒤 순위 최적화를 한다. 이 단계가 Exomiser·DeepRare 계열에는 없던 부분이다. (2) 이종 증거 타입의 병렬 통합: 규칙 기반 증거, 서열·변이 효과 표현, 표현형 조건부 생물학 맥락을 동시에 변이 점수에 반영한다. "LLM + 변이 도구 직렬 연결"이 아니라 데이터 기반 통합이 핵심이다.

새롭지 않은 점: LLM을 임상 유전체에 적용하는 에이전트 워크플로 자체는 선행 연구들이 이미 제안했다. DeepRare, Exomiser, prompted LLM reranking 모두 이 공간의 기존 주자들이다.

**핵심 가정 (깨지면 무너지는 전제)**

1. 자체 18,622건 코호트가 평가에 쓰이는 held-out 벤치마크와 독립적이라는 전제. 동일 기관에서 나온 데이터이므로 분포 이동(distribution shift) 없이 성능이 유지될 것이라는 암묵적 가정이 깔려 있다.
2. "해결된 케이스(solved cases)"만으로 구성된 내부 벤치마크가 실제 임상 혼합 코호트를 대표한다는 가정. 미해결 케이스를 제외하면 어렵고 모호한 사례가 빠져 성능이 과대평가될 수 있다.
3. 사전학습 단계의 대조학습이 다운스트림 순위 최적화에 유의미한 이점을 준다는 가정 — ablation으로 검증됐다고 보고하지만 세부 수치는 초록에 없다.

**재현성**

코드: 미공개. 논문 내 GitHub 링크 미확인.
데이터: 미공개. 18,622건 코호트는 자체 보유 임상 데이터.
하이퍼파라미터: 미제공 (초록 기준).
베이스라인 공정성: Exomiser, DeepRare, prompted LLM reranking 세 가지를 비교했으나, 각 베이스라인의 버전·프롬프트 설정이 초록에 명시되지 않아 공정성을 외부에서 검증하기 어렵다.
추정: 코드·데이터 비공개인 채로는 독립 재현이 사실상 불가능하다.

**검증 방식** (in silico, 내부 held-out)

- 내부 held-out 해결 사례 벤치마크에서 Recall@1/3/5/10 = 0.658 / 0.882 / 0.912 / 0.929 (초록 기준).
- 외부 데이터셋 검증: 미제공 (초록 기준). 다른 기관 코호트 결과 없음.
- Ablation: 규칙 기반 증거·기전 맥락·전문가 세분화 각각을 제거하는 분석을 수행했다고 보고. 세부 수치는 미제공.
- 통계적 유의성: 초록에 p값·신뢰구간 미기재.
- 검토필요: held-out 비율, 환자 수준 분리 여부(동일 환자의 다른 변이가 train/test에 섞이지 않았는지) 원문에서 확인 필요.

**강점**

- 18,622건 코호트는 이 분야 공개 벤치마크 대비 압도적으로 큰 실제 임상 데이터다.
- 사전학습된 증거 인코더 + 에이전트 레이어 조합은 확장 가능한 파이프라인 설계다.
- Recall@10 = 0.929은 실제 임상 운용에서 1차 스크리닝 도구로 쓸 수 있는 수준이다.

**한계**

- 단일 기관(Zhejiang University) 내부 데이터만 사용 — 외부 검증 없이 범용성 주장 어렵다.
- 미해결 케이스 제외 벤치마크 → 성능 낙관 편향 가능성.
- 코드·데이터 미공개 → 방법의 핵심 구현(증거 엔진 구조, 대조학습 세부)을 외부에서 검증할 방법이 없다.
- 희귀 변이 스펙트럼이 Zhejiang 코호트에 특화됐을 경우 다른 인종 집단에서 성능 저하 위험.

**내 연구 적용**

희귀 유전 질환 진단 파이프라인을 구성할 때 이 에이전트 워크플로의 설계 아이디어(증거 타입 병렬 통합 + 표현형 조건부 컨텍스트 주입)는 참고할 만하다. 단, 코드 공개 전까지 직접 차용은 불가하고, 유사한 구조를 자체 구현할 때 베이스라인으로 삼을 수 있다. 외부 검증 결과가 나오기 전까지는 성능 수치를 그대로 인용하지 않는다.

## 2. Stable-Shift: 미관찰 유전자 교란 전사 반응 예측

**arXiv**: https://arxiv.org/abs/2606.24940 (arXiv:2606.24940 · 2026-06-22 · **preprint, 미동료심사**)
**저자**: Sajib Acharjee Dip, Liqing Zhang | Virginia Tech, Computer Science
**코드**: https://github.com/Sajib-006/PerturbGraph (공개, ACM-BCB 2026 발표 예정)

**문제의식 선행 — 왜 이 질문이 필요했나**
Perturb-seq 스크린에서 실제로 교란한 유전자 수는 전체 유전자의 극히 일부다. 관찰하지 못한 유전자(unseen gene)에 대한 zero-shot 예측이 가능해야 드러그 타깃 스크리닝의 실용성이 열린다. 기존 GEARS 등은 학습 분포 내 교란에 강하지만 unseen 유전자 외삽에 체계적 약점이 있었다.

**무엇이 새로운가**

주장된 신규성: "교란 수준 발현 변화량(shift)으로의 집계 + 저랭크 반응 기저 구성 + 생물학 컨텍스트에서 새 유전자 좌표 예측"의 3단 구조.

실제 새로운 점: (1) 저랭크 기저 분해 접근 — 학습 교란들로 반응 공간의 주성분(basis)을 구성한 뒤, unseen 유전자는 그 기저에서 좌표만 예측한다. 이는 scGPT·GEARS의 end-to-end 임베딩 접근과 다른 귀납 편향이다. (2) 집계 단위의 설계 선택 — 단일세포 노이즈를 제거하기 위해 먼저 교란 수준(pseudo-bulk)으로 집계하고 이를 학습 대상으로 삼는다.

새롭지 않은 점: STRING + GO + GCN 조합 자체는 GEARS를 포함한 선행 연구들이 이미 활용했다. 저랭크 분해도 차원 축소의 변형이다. 이 논문의 기여는 둘을 '기저 공간 좌표 예측'이라는 프레임으로 묶은 설계에 있다.

**핵심 가정 (깨지면 무너지는 전제)**

1. 학습 교란들이 반응 공간의 충분한 기저를 포괄한다는 가정. 기저에 없는 방향의 반응이 unseen 유전자에서 나타나면 예측이 체계적으로 실패한다.
2. STRING 상호작용이 K562 세포 맥락에서도 유효하다는 가정. STRING은 조직·세포 유형 비특이적이므로 특정 세포주에서 오히려 노이즈가 될 수 있다.
3. 코사인 유사도가 "생물학적으로 의미 있는" 전사 반응을 충분히 포착한다는 가정. 방향만 맞고 크기가 틀린 경우도 높은 점수를 줄 수 있다.

**재현성**

코드: 공개 (https://github.com/Sajib-006/PerturbGraph). seed 42 고정, `run_benchmark_*.sh` 스크립트 포함, `requirements.txt`·`environment.yml` 제공.
데이터: K562 Perturb-seq 공개 벤치마크 사용 (CZI Science 플랫폼에 공개).
베이스라인 공정성: GEARS와 비교했으며 동일 데이터셋 사용 — 비교적 공정하다. 다만 scGPT, PERT 같은 더 최신 베이스라인이 빠졌을 가능성이 있다.
질문: GEARS 이후 등장한 foundation model 기반 교란 예측기(예: 2025~2026년 논문들)와의 비교가 필요하다.

**검증 방식** (in silico, 단일 데이터셋)

- K562 Perturb-seq 벤치마크: 코사인 유사도 0.592 (GEARS 0.569, 초록 기준).
- Spearman 상관·상위 유전자 정밀도에서도 비교 방법 중 최고라고 보고 (수치 미제공, 초록 기준).
- 외부 데이터셋 검증: 미제공. K562 단일 세포주로만 평가.
- 조합 교란(combinatorial perturbation) 조건 결과: 미확인 (초록에 명시 없음).
- Ablation: 초록에 명시 없음. 원문 확인 필요.

**강점**

- 코드·데이터 공개로 재현 가능성이 이번 주 5편 중 가장 높다.
- 저랭크 기저 설계는 해석 가능성(어떤 반응 방향에 기여하는지)을 열어 둔다.
- K562 표준 벤치마크에서 GEARS 대비 코사인 유사도 +4.0%p 향상 (0.569 → 0.592, 초록 기준).

**한계**

- K562 단일 세포주 벤치마크만 사용 — iPSC, 일차세포 등 다른 맥락에서의 성능이 미지수.
- 코사인 유사도 개선폭(+0.023)이 실제 생물학적 의미에서 유의한지 별도 평가 필요.
- 조합 교란(2-유전자 이상 동시 KO) 예측으로의 확장 가능성이 초록에서 논의되지 않음.
- STRING 그래프가 실험 세포 맥락을 왜곡할 경우 오히려 성능 저하 가능 — ablation 여부 미확인.

**내 연구 적용**

Perturb-seq 데이터를 다루거나 드러그 타깃 교란 반응을 in silico로 스크리닝할 때 기저 분해 접근을 파이프라인에 통합하는 방안을 검토할 수 있다. 코드가 공개돼 있어 바로 테스트 가능하다. 단, K562 단일 벤치마크라는 점을 감안해 자체 세포주·오가노이드 데이터로 전이 가능성을 먼저 확인해야 한다.

## 3. Molexar: 통합 다중모달 분자 파운데이션 모델

**arXiv**: https://arxiv.org/abs/2606.25865 (arXiv:2606.25865 · 2026-06-24 · **preprint, 미동료심사**)
**저자**: Haoyu Lin, Yiyan Liao 외 | Peking University (Peking-Tsinghua Center for Life Sciences)

**문제의식 선행 — 왜 이 질문이 필요했나**
분자 생성 모델들이 특성 예측, 구조 기반 설계, 파마코포어 기반 최적화를 각각 별도 모델로 처리해왔다. 이는 멀티태스크 학습의 이점을 버리고, 조건 유형마다 추론 파이프라인을 따로 관리해야 하는 부담을 낳는다. 단일 자기회귀 경로로 모든 조건을 통합할 수 있는가라는 물음이 이 논문의 출발점이다.

**무엇이 새로운가**

주장된 신규성: Fragment-SELFIES 기반 단일 자기회귀 디코더가 스칼라 특성·파마코포어·단백질 서열·결합 포켓 등 이종 조건을 모두 처리.

실제로 새로운 점 두 가지. (1) Fragment-SELFIES 언어: BRICS 단편(fragment) 트리 구조를 토큰 문법으로 직접 인코딩해 유효성(validity)을 구조적으로 보장한다. 기존 SELFIES가 원자 수준 유효성을 보장했다면, Fragment-SELFIES는 그것을 단편 수준으로 끌어올린 것이다. (2) 값-토큰 임베딩 인플레이스 교체(in-place replacement): 서로 다른 조건 유형을 별도 브랜치나 어댑터 없이 동일한 위치에 임베딩을 교체해 주입한다. 이로써 사전학습 자기회귀 경로를 그대로 재사용하며 SFT만으로 조건부 생성을 획득한다.

새롭지 않은 점: 단일 모델로 다중 조건 분자 생성을 통합하려는 시도는 MolGen, DrugGEN 등 선행 연구가 있다. Gemma2 스타일 디코더 자체는 공개 모델의 재사용이다.

**핵심 가정 (깨지면 무너지는 전제)**

1. Fragment-SELFIES 단편 공간이 약물 설계에 필요한 화학 공간을 충분히 커버한다는 가정. BRICS 단편화는 특정 결합 패턴에만 적용되므로 매크로사이클·천연물 구조에서 표현력이 제한될 수 있다.
2. 동일한 자기회귀 경로가 이질적 조건(수치 특성 vs. 3D 포켓 기하)을 모두 인코딩하기에 충분하다는 가정. 정보 타입이 다를 때 단순 임베딩 교체가 충분한지는 ablation이 관건이다.
3. SFT 단계의 조건-분자 쌍 데이터 품질이 생성 품질을 결정짓는다는 가정 — 데이터 편향이 생성 분자에 직접 반영된다.

**재현성**

코드: 미공개. arXiv 페이지와 검색 결과에서 공식 GitHub 링크 미확인. (GitHub 계정 "Molexar"가 존재하나 본 논문 코드와의 연결 미확인.)
데이터: CrossDocked2020은 공개 벤치마크. MolGenBench는 초록에서 이름만 언급 — 해당 벤치마크의 세부 구성을 원문에서 확인해야 한다.
베이스라인: 초록에 베이스라인 이름이 명시되지 않음. "경쟁력을 유지한다"는 표현만 있어 수치 비교 내용은 원문 확인 필요.
추정: 코드 미공개 상태로는 Fragment-SELFIES 구현·사전학습 데이터·SFT 데이터 구성을 재현할 수 없다.

**검증 방식** (in silico)

- 비조건부 생성: 유효성(validity) 100%, 높은 약물 유사성(drug-likeness) — 초록 기준. 구체적 QED·SA score 수치 미제공.
- CrossDocked2020 테스트 세트: 목표 조건부 생성 "경쟁력 유지" — 수치 미제공, 초록 기준.
- MolGenBench: 안전성·효능 지표에서 유리한 결과 — 수치 미제공, 초록 기준.
- Ablation: 초록에 명시 없음.
- 외부·생물학적 검증: 없음. 전적으로 in silico.
- 검토필요: 초록에 구체적 수치가 거의 없어 실제 성능 격차를 현재 단계에서 판단하기 어렵다.

**강점**

- 단일 모델로 구조 기반(포켓), 리간드 기반(파마코포어), 특성 조건 생성을 모두 처리하는 설계 아이디어는 추론 파이프라인 단순화 측면에서 실용적이다.
- Fragment-SELFIES의 유효성 100% 달성은 생성 모델에서 유효하지 않은 분자를 후처리로 제거해야 하는 부담을 없앤다.
- Gemma2 기반의 소형 파라미터로 대형 모델과 경쟁력을 가진다고 보고.

**한계**

- 초록에 구체적 수치가 거의 없어 주장된 성능을 검증할 수 없다 — 코드·원문이 공개되기 전까지 claim 수용에 신중해야 한다.
- BRICS 단편화 기반의 Fragment-SELFIES는 매크로사이클·PPI 억제제·공유결합 억제제 같은 비전통적 화학 공간에서 표현력 한계가 있을 수 있다.
- 분자 생성 유효성 100%는 SFT 데이터 분포 내에서의 수치이므로 신규 스캐폴드(scaffold) 생성 다양성은 별도 평가 필요.
- 단백질 서열 조건 생성이 3D 구조 정보를 실제로 활용하는지, 아니면 서열 수준 패턴에 의존하는지 불명확하다.

**내 연구 적용**

신약개발 hit-to-lead 단계에서 다중 조건(결합 포켓 + ADMET 특성 + 파마코포어)을 동시에 걸어야 하는 설계 문제에 이 통합 접근이 이론적으로 맞다. 그러나 코드 공개 전까지는 관망이 합리적이다. Fragment-SELFIES 표현 방식 자체는 공개 구현 여부와 무관하게 내 데이터에 맞는 분자 언어를 선택할 때 고려할 수 있다.

## 4. 유전적 증거와 신약 승인 연관성: 26,278 타깃-질환 쌍 관찰 연구

**arXiv**: https://arxiv.org/abs/2606.14823 (arXiv:2606.14823 · 2026-06-12 · **preprint, 미동료심사**)
**저자**: Victoria Paterson | University of Edinburgh

**기존 한계 → 이번 해결**
기존 유사 연구들(Minikel et al. 2021, 선행 Open Targets 분석)은 문헌 마이닝 기반 증거가 승인 이후 발간된 논문을 포함하는 시간적 누출(temporal leakage) 문제를 제대로 통제하지 못했다. 이 논문은 (1) 2015년 기준 시간 분할 검증, (2) 6가지 증거 유형 각각을 제거하는 특성 제거 분석을 도입해 어떤 증거 유형이 실제 예측력을 제공하는지와 어떤 것이 누출 신호인지를 분리하려 했다.

**무엇이 새로운가**

주장된 신규성: 대규모 타깃-질환 쌍(26,278) + 시간적 검증 + 특성 제거 분석의 조합.

실제 새로운 점: 특성 제거 분석에서 문헌 마이닝(literature mining) 단독이 분류기 성능의 대부분(AUPRC 0.099 vs. 전체 0.109)을 설명한다는 것을 명시적으로 보고한 점이다. 이는 기존 연구들이 "유전적 증거가 신약 승인을 예측한다"고 주장할 때 실제로는 문헌 마이닝이라는 누출 채널에 의존했을 가능성을 제기하는 방법론적 기여다.

새롭지 않은 점: Open Targets + ChEMBL을 기반으로 타깃-질환 쌍을 구성하는 분석 자체는 선행 연구들이 해온 접근이다. 로지스틱 회귀 분류기 역시 새롭지 않다.

**핵심 가정 (깨지면 무너지는 전제)**

1. 2015년 시간 분할이 "시간적 누출"을 충분히 통제한다는 가정. Open Targets 데이터 자체의 생성 시점이 명확히 분리되지 않는다면 누출이 완전히 제거되지 않을 수 있다.
2. 타깃 수준 분석에서 부트스트랩 CI가 동일 유전자 공유로 인한 비독립성을 충분히 보정한다는 가정.
3. 관찰 연구이므로 유전적 증거와 승인 사이에 인과관계가 아닌 교란 변수(confounder)가 OR을 부풀릴 가능성이 항상 존재한다.

**재현성**

코드: 미확인 (초록 기준). Open Targets·ChEMBL은 공개 DB이므로 데이터 접근 자체는 가능하다.
데이터: 공개 DB(Open Targets, ChEMBL) — 재현 분석 원칙적으로 가능.
분석 방법: 로지스틱 회귀, 특성 제거 분석, 부트스트랩 CI — 표준 통계 방법으로 비교적 재현 가능성 높음.
추정: 코드가 공개된다면 재현 시도가 가능한 몇 안 되는 논문 중 하나다.

**검증 방식** (관찰, 대규모)

- 타깃-질환 쌍 26,278건: 유전적 연관성 있는 타깃 vs. 없는 타깃, 승인률 OR = 3.25 (95% CI 2.79–3.79, p = 1.91×10⁻⁴²) — 초록 기준.
- 타깃 수준 보정 분석: OR = 2.79 (부트스트랩 95% CI 2.22–3.53) — 초록 기준.
- 시간적 검증 (2015년 이후 승인): OR = 3.51, p = 1.72×10⁻⁸ — 초록 기준.
- 특성 제거 분석: 문헌 마이닝 AUPRC 0.099 vs. 전체 특성 AUPRC 0.109 — 초록 기준.
- 외부 코호트 검증: 해당 없음 (단일 데이터소스 관찰 연구).
- 인과 추론: 없음. 관찰 연구 한계 내에 머뭄.

**강점**

- 대규모 데이터 + 방법론적 정직성: 문헌 마이닝 누출을 스스로 발굴·보고한 점이 신뢰도를 높인다.
- 시간적 분리 검증으로 "미래 데이터를 과거 예측에 쓴 게 아닌가"라는 비판에 일정 수준 대응했다.
- 공개 DB 기반이라 방법론을 독립적으로 따라 해볼 수 있다.

**한계**

- 관찰 연구이므로 인과 추론 불가. "유전적 증거가 있으면 승인 가능성이 높다"는 연관성이지 "유전적 증거를 갖추면 성공률이 올라간다"는 처방은 아니다.
- 문헌 마이닝 누출이 부분적으로 잔존할 가능성 — 2015년 시간 분할이 Open Targets 내 모든 데이터 필드에 동일하게 적용됐는지 원문 확인 필요.
- 치료 영역(therapeutic area)별 이질성이 통합 OR에 가려질 수 있다. 암·심혈관·희귀 질환은 타깃 발굴 패러다임이 다르다.
- 코드 미공개(현재 기준) — 완전한 재현 검증 미실시.

**내 연구 적용**

타깃 우선순위화 프로젝트에서 "유전적 증거 유형별 실제 예측력"을 정량화하는 분석 틀로 직접 참고 가능하다. 특히 문헌 마이닝 vs. GWAS vs. 희귀 변이 증거를 분리 평가하는 특성 제거 프레임워크는 내부 타깃 스코어링 시스템을 설계할 때 누출을 방지하는 방법으로 쓸 수 있다. 단, 결과를 인과적으로 해석하지 않도록 주의한다.

## 5. 범용 LLM이 전문 임상 AI 도구를 능가: 의료 벤치마크 비교 연구

**DOI**: https://doi.org/10.1038/s41591-026-04431-5 (Nature Medicine · 2026-06-12 · **peer-reviewed**)
**저자**: Vishwanath K., Alyakin A. 외 | NYU Langone Health / NYU Grossman School of Medicine
**코드**: https://github.com/nyuolab/clinical-llm-benchmarks (공개, GNU AGPL v3)

**기존 한계 → 이번 해결**
FDA 승인 임상 AI 도구와 범용 프론티어 LLM을 공정한 조건에서 나란히 비교한 동료심사 연구가 부재했다. 업체 주도 벤치마크는 자사 도구에 유리한 평가 설계를 쓸 유인이 있다. 이 논문은 제3자 연구팀이 맹검 임상의 평가를 설계해 비교한 첫 동료심사 연구 중 하나다 (외부 맥락: 선행 비교 연구들은 단일 기관 소규모이거나 자체 벤치마크에 한정된 경우가 많았다).

**무엇이 새로운가**

주장된 신규성: MedQA·HealthBench·RCQ(실제 임상 쿼리) 3단 평가 + FDA 승인 임상 AI 도구와 범용 LLM의 동시 비교.

실제로 새로운 점: RCQ(Real Clinical Query) 벤치마크 설계가 핵심이다. 실제 임상 환경에서 의사가 LLM에 제출한 100건의 익명화된 쿼리를 12명 미국 임상의가 맹검·무작위 평가해 모델당 1,800건 주석을 생성했다. 이는 시험 문제 기반 평가(MedQA)와 달리 실세계 임상 쿼리 분포를 반영한다는 점에서 방법론적 강점이다.

새롭지 않은 점: MedQA·HealthBench는 기존 공개 벤치마크다. GPT vs. 임상 AI 비교도 여러 선행 연구가 있다. 이 논문의 새로움은 세 단계를 한 논문에서 통합하고 동료심사를 통과한 데 있다.

**핵심 가정 (깨지면 무너지는 전제)**

1. 12명 임상의 패널이 미국 전체 임상의 판단을 대표한다는 가정. 전문과 분포, 경험 연차가 패널에 따라 결과가 달라질 수 있다.
2. 100건 RCQ가 실제 임상 쿼리 공간을 대표한다는 가정. 특정 전문과·증례 유형으로 편향됐을 가능성.
3. temperature=0.0 (결정론적) 생성 설정이 임상 사용 환경을 대표한다는 가정 — 실제 임상에서는 다양한 설정으로 사용된다.
4. OpenEvidence·UpToDate Expert AI가 이 비교에서 "최선의 설정"으로 운용됐다는 가정 — 도구 특성에 맞는 최적 사용법이 적용됐는지 확인 필요.

**재현성**

코드: 공개 (https://github.com/nyuolab/clinical-llm-benchmarks). 맹검 평가 Next.js 인터페이스, MedQA·HealthBench 스코어링 파이프라인 포함.
데이터: 부분 공개. MedQA·HealthBench는 공개. RCQ 100건은 "개인 임상 쿼리 포함 가능" 사유로 미공개. OpenEvidence·UpToDate Expert AI 출력은 제공되지 않음.
하이퍼파라미터: temperature=0.0, seed=62 명시. 검색(retrieval) 활성화 설정.
완전 재현 한계: RCQ 쿼리와 일부 모델 출력이 없어 완전 재현은 어렵다. MedQA·HealthBench 파트는 API 키가 있으면 재현 가능.

**검증 방식** (비교 연구, 전향적 설계 요소 포함)

- MedQA 500문항 (in silico 유형, 표준화된 의학 지식 평가).
- HealthBench 500항목 (임상의 정렬도).
- RCQ 100건 × 12명 임상의 맹검 평가 = 1,800건 모델-질문 주석 (실제 임상 쿼리 기반, 가장 실세계에 가까운 평가).
- 세 단계 모두에서 GPT-5.2·Gemini 3.1 Pro·Claude Opus 4.6이 OpenEvidence·UpToDate Expert AI 상회 (수치 미제공, 초록·보도 기준).
- 임상 AI 도구들은 RCQ에서 Google Search AI Overview와 동등 수준 — 초록 기준.
- 통계 검정 방법·신뢰구간: 초록에 미기재. 원문 확인 필요.
- 외부 기관 검증: 없음. NYU Langone 단일 기관 설계.

**강점**

- 3단 평가 중 RCQ는 실세계 유효성에 가장 가까운 설계다.
- 코드 공개로 방법론 투명성 확보.
- 동료심사 Nature Medicine 게재로 선행 리뷰 통과.
- 맹검·무작위 설계로 임상의 편향 최소화 시도.

**한계**

- RCQ 100건은 통계적 검정력이 제한적이다. 전문과·증례 편향이 있을 경우 결론이 달라질 수 있다.
- FDA 승인을 받은 임상 AI 도구들이 이 벤치마크에서 "최선으로 사용됐는지" 독립 검증 어렵다. 도구 측에서 이의 제기가 가능하다.
- 프론티어 LLM들은 이 논문 발표 이후에도 빠르게 업데이트되므로, 이 비교는 특정 시점 스냅샷이다.
- 임상 AI 도구의 핵심 차별점(안전 가드레일, 출처 인용, 규제 순응성)이 이 평가 지표(정확성·완전성·안전·명확성)에 완전히 반영됐는지 불분명하다.
- 질문: NYU 환경의 임상 쿼리가 다른 의료 시스템(한국, 유럽 등 다국어·다기관)에서도 동일한 격차를 보일지 별도 검증 필요.

**내 연구 적용**

바이오인포 연구에 범용 LLM을 쓸 때 "전문화된 도구가 더 낫다"는 직관을 검토하게 만드는 결과다. 내부 파이프라인에서 임상 질의응답 기능을 구현할 때 전문 도구에 먼저 투자하기보다 범용 LLM을 기준선으로 먼저 테스트하는 설계가 합리적임을 지지한다. 다만 이 결과를 한국어 임상 환경이나 바이오인포 특화 쿼리로 일반화할 때는 별도 검증이 필요하다.
