## 논문 1. scGTN: 단일세포 RNA-seq 클러스터링 샴 그래프 트랜스포머

**원문**: https://arxiv.org/abs/2606.18672
**출처**: arXiv (preprint, 미동료심사) · 2026-06-17
**검증 수준**: in silico

### 임상적 질문

"단일세포 RNA 시퀀싱(scRNA-seq) 데이터에서 세포 유형(cell type)을 더 정확하게 클러스터링할 수 있는가." 이 논문이 직접 다루는 것은 방법론 문제다. scRNA-seq 데이터가 가진 희소성(sparsity)과 높은 잡음, 그리고 세포 간 구조 정보가 기존 방법에서 충분히 활용되지 않는 한계를 지적하며, 샴 그래프 트랜스포머(Siamese Graph Transformer)로 이를 해결하고자 한다.

임상 미충족 수요와의 접점: `외부 맥락:` 세포 유형 분류의 정확도는 종양 미세환경(tumor microenvironment) 분석, 희귀 세포 아집단(rare subpopulation) 발굴, 약물 반응성 예측 등 다운스트림 임상 연구의 기반이 된다. 그러나 이 논문 자체는 임상 결과와 연결된 분석을 수행하지 않으며, 클러스터링 정확도 개선 자체가 목적이다.

### 검증 수준

**in silico.** 공개 벤치마크 scRNA-seq 데이터셋 복수에서 평가. NMI(Normalized Mutual Information), ARI(Adjusted Rand Index) 등의 클러스터링 지표로 기존 방법 대비 우위를 보고한다.

구체 수치:
- "extensive experiments on multiple benchmark scRNA-seq datasets demonstrate scGTN consistently outperforms existing methods" — 원문 보고 표현
- `미제공:` 구체적인 데이터셋 이름, 각 데이터셋의 N수, NMI/ARI 절대값 수치는 초록에 기재되지 않아 본문 접근 없이 확인 불가
- `미제공:` 비교 대상 베이스라인 방법 목록과 각 수치도 초록 기준 미확인

`해석:` scRNA-seq 클러스터링 벤치마크는 레이블이 붙은 공개 데이터셋에서의 성능 비교가 표준 평가 방식이다. 이는 계산 검증이며 임상 코호트 검증과 다른 수준이다.

### 임상 적용까지의 거리

매우 멀다. 이 논문의 직접 임상 접점은 없으며, 잠재적 경로는 아래와 같다.

1. **다운스트림 연구 기여**: 개선된 클러스터링 방법이 종양 면역 세포 구성 분석, 단세포 수준 세포 상태 정의 등 기초·중개 연구에 활용될 수 있다
2. **임상 유용성으로의 거리**: 세포 유형 분류 정확도 개선 → 생물학적 발견 → 임상 마커 개발 → 임상 검증의 각 단계가 별도 연구를 요구한다
3. **규제**: 단일세포 데이터 분석 도구 자체는 현재 직접 규제 대상이 아니나, 이를 진단 목적에 쓰려면 임상 검증과 체외진단기기(IVD) 경로가 필요하다
4. `미제공:` 코드 공개 여부가 초록에 명확히 기재되지 않아 재현 가능성 판단 보류

### 수요자 관점

| 수요자 | 의미 |
|--------|------|
| **환자** | 직접 영향 없음. 장기적으로 세포 유형 분류 정확도 향상이 종양 이질성 분석의 기반이 될 수 있다 |
| **임상의** | 현 단계에서 직접 쓸 수 없음. scRNA-seq 분석이 임상에 들어오는 시점에 정확한 세포 분류 도구로서 간접 기여 가능 |
| **연구자·검사실** | 멀티오믹스 세포 유형 분류 파이프라인에 즉시 통합 시도 가능. 성능이 재현되면 유용한 도구 |
| **지불자** | 현 단계에서 급여 논의와 무관 |

### 위험·한계

- **임상 관련성 부재**: 클러스터링 정확도와 임상 결과 사이의 연결고리를 이 논문은 제시하지 않는다. ARI·NMI 개선이 생물학적·임상적 발견으로 이어지는지는 별개 검증이 필요하다
- **수치 불투명**: 구체적 성능 수치가 초록에 없어 독립적 평가 불가. 프리프린트 단계에서 중요한 제약
- **데이터 대표성**: 벤치마크 데이터셋이 특정 조직·기술·배치 조건에 편향되어 있을 수 있으며, 이 한계가 모델 평가에 반영됨
- **과대해석 경계**: "consistently outperforms"는 선택된 벤치마크에서의 결과이며, 다른 데이터 분포·시퀀싱 플랫폼·배치 효과 환경에서의 일반화는 `미제공:`

## 논문 2. Probing+Fusion 암 FM: 멀티모달 암 분석 파운데이션 모델 체계적 평가

**원문**: https://arxiv.org/abs/2606.17115
**출처**: arXiv (preprint, 미동료심사) · 2026-06-15
**검증 수준**: in silico / 후향적 상업 코호트 (외부 검증 없음)

### 임상적 질문

"전체 슬라이드 이미지(WSI)와 전사체 프로파일을 동시에 활용하는 파운데이션 모델이 암 진단·분류 과제에서 신뢰할 수 있는 성능을 보이는가." 구체적으로 세 가지 하위 질문을 던진다: (1) 이미지·오믹스 두 단일모달 표현 중 어느 쪽이 어떤 과제에 더 강한가, (2) 멀티모달 융합이 단일모달 대비 일관된 이점을 주는가, (3) 분포 이동(distribution shift) 하에서 모델의 신뢰성은 어떠한가.

이 질문의 임상적 배경: 병리 파운데이션 모델이 임상에 배포되려면 연구 환경과 실제 임상 코호트 간 분포 이동에서도 예측이 신뢰할 수 있어야 한다.

### 검증 수준

**in silico / 후향적 상업 코호트** (외부 검증 없음).

코호트:
- **IH-BC** (유방암): Flatiron Health-Caris Life Sciences 임상-분자 데이터베이스(CMDB)에서 추출한 미국 내 탈식별화 코호트. WSI + 전사체 쌍 포함
- **IH-NSCLC** (비소세포폐암, Non-Small Cell Lung Cancer): 동일 데이터베이스 원천
- `미제공:` 두 코호트 각각의 정확한 환자 수(N)는 초록에 기재되지 않음
- 8개 하위 분류 과제에서 5개 파운데이션 모델 평가 (세 가지 이미지-오믹스 융합 전략 비교)

수치:
- 이미지 파운데이션 모델들이 여러 과제에서 "broadly comparable" 성능, AUC 최소값 기준 0.6478 이상
- 유방암 과제에서 등각 예측(conformal prediction) 집합이 2~3 클래스 내 수렴, 폐암 세부 구분 과제보다 신뢰도-효율성 균형이 우호적
- `미제공:` 개별 과제별 정확한 AUC/Accuracy 수치, 기저 임상 모델과의 비교, 환자 수 세부 사항
- `미제공:` 어떤 5개 파운데이션 모델인지 초록 기준 확인 불가 (본문 접근 필요)

`해석:` AUC 0.6478은 단일 수치이며 8개 과제 중 어느 과제의 어느 모델 결과인지 맥락 없이는 해석 불가. 이 수치를 전체 성능의 대표치로 읽는 것은 과대해석이다.

### 임상 적용까지의 거리

중간~멀다. 이 논문은 적용 논문이 아니라 배포 전 평가(pre-deployment evaluation) 논문이며, 그 자체로 중요한 위치다.

1. **현재 기여**: 임상 배포 전 파운데이션 모델 선택과 신뢰성 점검에 방법론적 틀을 제공한다. 분포 이동 하 등각 예측은 실용적 신뢰성 지표다
2. **남은 단계**:
   - 전향적 임상 코호트 또는 다기관 외부 검증이 없다
   - 분류 과제(LOH 상태, 분자 아형 등)가 임상 의사결정에 직접 연결되는지 별도 검증 필요
   - `외부 맥락:` 계산 병리학(computational pathology) 도구가 실제 임상 워크플로에 들어오려면 FDA의 SaMD(Software as a Medical Device) 또는 IVD 경로를 통한 임상 검증이 요구된다
3. **규제**: 암 분류 보조 도구로의 활용에는 전향 임상시험 근거가 필요하다

### 수요자 관점

| 수요자 | 의미 |
|--------|------|
| **환자** | 직접 영향 없음. 병리 AI 도구의 신뢰성 기준을 올리는 데 간접 기여 |
| **임상의 (병리의사)** | 파운데이션 모델이 어떤 과제에서 믿을 수 있고 어디서 과신해서는 안 되는지 근거를 제공한다. 즉시 임상 적용은 아니나 기술 평가에 유용 |
| **연구자·AI 개발자** | 멀티모달 융합 전략 선택과 신뢰성 평가 방법론으로 즉시 참조 가능 |
| **지불자·규제기관** | 파운데이션 모델 기반 암 분류 도구의 임상 승인 요건 논의에서 이런 체계적 평가 방법론이 기준 틀이 될 수 있다 |

### 위험·한계

- **코호트 비공개**: IH-BC, IH-NSCLC는 상업 데이터베이스(Flatiron-Caris) 기반이며 외부 재현이 어렵다. 규모와 구성에 대한 상세 정보 `미제공:`
- **외부 독립 검증 없음**: 동일 원천 코호트 내 벤치마크로, 진정한 분포 이동 저항성을 검증하려면 별도 기관 데이터가 필요하다
- **과제 선택 편향**: 8개 분류 과제가 어떤 기준으로 선택됐는지, 파운데이션 모델 개발자의 이해충돌 `미제공:` — `검토필요:` 저자 중 Genentech 소속 확인 시 이해충돌 명시 여부 검토
- **멀티모달 융합 이점의 일관성**: 초록에서 멀티모달 융합이 "추가 이득을 주는가"를 검토한다고 기술하나, 그 결론이 긍정적인지 부정적인지 초록에서 명확히 `미제공:`

## 논문 3. Fair Concordance xCI: 그룹 조건부 C-지수로 생존 예측 공정성 평가

**원문**: https://arxiv.org/abs/2606.16872
**출처**: arXiv (preprint, 미동료심사) · 2026-06-15
**검증 수준**: 후향 (Framingham/MESA/ARIC 조화 데이터 및 Truveta EHR)

### 임상적 질문

"생존 분석 예측 모델이 인구통계 집단 간에 공정하게 작동하는가." 기존 하렐 C-지수(Harrell's Concordance Index)는 전체 집단에 대한 단일 지표로, 특정 인종·성별·연령 집단에서 예측 성능이 떨어지는 편향을 숨긴다. 이 논문은 이 공정성 감사(fairness audit) 문제를 형식화하고 측정 도구를 제안한다.

임상 미충족 수요와의 접점: 매우 직접적이다. 심혈관 질환(CVD) 위험 예측 모델은 이미 임상에서 쓰이고 있다(예: Pooled Cohort Equations). `외부 맥락:` 이런 모델들이 흑인·라틴계·여성 등 일부 집단에서 예측 정확도가 낮다는 지적이 의료 AI 공정성 연구에서 반복적으로 제기되어 왔다. xCI는 이 문제를 공식 지표로 정량화한다.

### 검증 수준

**후향** (N=미제공, 코호트 다수).

코호트:
- **케이스 스터디 1**: Framingham Offspring, MESA, ARIC 세 코호트의 조화(harmonized) 데이터 — CVD 위험 예측 생존 모델 공정성 평가
- **케이스 스터디 2**: Truveta 대규모 EHR 데이터베이스 — 기존 CVD 위험 예측 모델의 집단 간 공정성 평가
- `미제공:` 각 코호트의 정확한 N수, 분석에 포함된 인구통계 변수(인종, 성별, 연령 구분 방식)의 세부 사항

방법론 타당성:
- xCI는 Harrell C-지수를 집단 조건부로 확장하고, 역확률 검열 가중치(IPCW, Inverse Probability of Censoring Weighting)를 적용한 일치 추정량을 개발
- C-지수가 xCI들의 가중 평균임을 수학적으로 증명
- "xCI effectively detects biases across demographic groups that are overlooked by existing metrics" — 원문 보고

### 임상 적용까지의 거리

가깝다 (도구로서). 단, 임상 결과 개선과는 별개다.

1. **즉각 활용 가능**: xCI는 기존에 배포된 생존 모델의 공정성 감사 도구로 즉시 쓸 수 있다. 새 임상 모델 개발·평가 시 xCI를 추가 평가 지표로 도입하는 데 기술적 장벽이 낮다
2. **규제적 활용**: `외부 맥락:` FDA는 AI/ML 기반 소프트웨어 의료기기(SaMD)의 안전성 평가에 공정성 지표 포함을 검토하고 있다. xCI는 이 맥락에서 실용적 도구다
3. **임상 결과 개선까지의 거리**: xCI가 편향을 감지하는 것과, 그 편향을 수정해 실제 환자 결과가 개선되는 것은 별개다. 후자는 추가 개입 연구가 필요하다
4. `추정:` 공정성 감사 도구로서의 채택에는 학계·규제기관의 검토와 표준화 과정이 필요하다

### 수요자 관점

| 수요자 | 의미 |
|--------|------|
| **환자 (특히 소수집단)** | 자신의 집단에서 예측 모델이 얼마나 정확한지 알 수 있는 근거가 생긴다. 직접적 혜택은 모델 개선 이후 단계 |
| **임상의** | CVD 위험 예측 도구를 다양한 환자 집단에 적용할 때 신뢰성의 집단별 한계를 인식하는 데 도움 |
| **AI 개발자·연구자** | 생존 모델 공정성 평가의 표준 도구로 즉시 채택 가능. IPCW 기반 추정량은 기존 생존 분석 파이프라인에 통합 용이 |
| **지불자·규제기관** | AI 기반 예후 도구의 집단 간 형평성 요건 설정에서 xCI가 평가 기준이 될 수 있다 |

### 위험·한계

- **공정성 정의의 한계**: xCI는 예측 순위의 집단 간 차이를 측정하나, 공정성의 정의(기회 균등, 보정, 예측 동등 등)는 다원적이다. xCI가 모든 공정성 개념을 포괄하지는 않는다
- **집단 정의 의존성**: 어떤 집단 변수를 기준으로 xCI를 계산하느냐에 따라 결과가 달라진다. 집단 경계 설정이 분석의 핵심 결정이며, 이는 임상 맥락 이해 없이는 표준화하기 어렵다
- **후향 검증의 한계**: Framingham, MESA, ARIC는 모두 후향적 관찰 코호트다. xCI가 전향 설계에서도 동일하게 작동하는지, 그리고 편향 감지가 임상 결과 차이와 실제로 연결되는지는 `미제공:`
- **코드/데이터 비공개**: 재현 가능성 제한 (`미제공:` 공개 여부 초록 미기재)
- **과대해석 경계**: xCI로 편향이 감지된다고 해서 그 편향의 원인(데이터 부족, 모델 설계, 생물학적 차이 등)이 자동으로 밝혀지거나 해결되는 것은 아니다

## 논문 4. Clinical EHR Reasoning Failure: 임상 EHR QA에서 트랜스포머 합성 추론 한계 실증

**원문**: https://arxiv.org/abs/2606.16890
**출처**: arXiv (preprint, 미동료심사) · 2026-06-15 · UCSF Department of Medicine
**검증 수준**: in silico (MedAlign 벤치마크 데이터, N=313 QA 쌍)

### 임상적 질문

"임상 의사결정 지원에 사용되는 대형 언어 모델(LLM)이 EHR에서 여러 단계 추론이 필요한 임상 질문에 얼마나 신뢰할 수 있는가." 구체적으로, 추론 단계 수(hop count)가 늘어날수록 LLM 정확도가 체계적으로 저하되는지를 실증한다.

이것은 임상 AI 안전성(clinical AI safety)의 핵심 질문이다. LLM이 EHR 기반 의사결정 지원에 도입될 때, 어떤 종류의 질문에서 실패하는지 사전에 알아야 한다.

### 검증 수준

**in silico / 벤치마크 평가.** 실제 임상 결과와의 연결이 없는 QA 정확도 측정이다.

데이터:
- **MedAlign**: 임상의가 생성한 313개 EHR QA 쌍 (공개 벤치마크). 비정형 개방형 질문과 EHR XML 기록 포함
- 313개 중 301개가 hop count 주석 완료 후 평가에 사용됨
- 4단계 hop 분류(hop=1~4)로 주석화

수치:
- **Claude Sonnet (claude-sonnet-4-6), zero-shot**: hop=1에서 30.6%, hop=4에서 17.6% (단조 감소)
- **Claude Sonnet, extended thinking (사고 확장 모드)**: 동일 within-model ablation 포함
- **GPT-4o, zero-shot**: 동일 단조 감소 패턴 확인
- **gpt-5.4-2026-03-05, zero-shot**: 동일 패턴 확인 (두 세대 GPT에서 재현)
- 세 모델, 두 공급사(Anthropic·OpenAI), 두 세대(GPT-4, GPT-5)에서 모두 재현 — 원문 보고

`해석:` hop=1에서 최고 모델도 정확도 30.6%라는 수치가 주목할 만하다. 절대 정확도 자체가 낮다는 점은 임상 배포에서 심각한 신호다. 단순 질문에서도 두 번에 한 번 이상 오답이 나온다.

`해석:` 확장 사고(extended thinking) 모드가 개선을 주는지 여부가 within-model ablation으로 확인된다고 기술되어 있으나, 그 개선폭이 얼마인지는 초록에서 `미제공:`

### 임상 적용까지의 거리

이 논문은 "적용 가능성"을 보이는 논문이 아니라 "적용 한계를 경고"하는 논문이다. 임상 AI 배포 논의에서 즉각 참조해야 할 안전성 근거다.

1. **즉각적 함의**: EHR 기반 임상 의사결정 지원 도구를 도입할 때, 다단계 추론이 필요한 질문(진단 추론, 복합 처방 검토, 다약제 상호작용 등)에서 LLM 단독 판단을 믿어서는 안 된다는 실증 근거를 제공한다
2. **한계**: MedAlign은 313개 QA 쌍으로 구성된 소규모 영어 임상 벤치마크다. 한국어·다국어 EHR, 국내 임상 환경에서의 일반화는 `미제공:` 검증이 별도로 필요하다
3. **규제적 함의**: `외부 맥락:` FDA의 임상 의사결정 지원 소프트웨어 가이드라인에서 LLM 기반 도구의 평가 요건이 논의 중이다. 이 논문 방법론(hop-count taxonomy)이 평가 틀의 후보가 될 수 있다

### 수요자 관점

| 수요자 | 의미 |
|--------|------|
| **환자** | 직접 의미: EHR 기반 LLM 의사결정 지원이 복잡한 임상 질문에서 신뢰할 수 없다는 근거가 생겼다. 즉 임상 배포에 신중해야 한다는 신호 |
| **임상의** | LLM을 임상 질의응답 보조로 쓸 때, 다단계 추론 질문에서 독립 판단을 맡기면 위험하다는 실증. 구체적 실패 구조를 알면 사용 범위를 현명하게 제한할 수 있다 |
| **병원·AI 배포 담당자** | hop-count taxonomy를 임상 AI 평가 기준으로 채택하면 배포 전 안전성 점검에 즉시 활용 가능 |
| **지불자·규제기관** | LLM 기반 임상 지원 도구의 보험급여·인허가 심사에서 추론 복잡도별 성능 요건 설정에 근거 제공 |

### 위험·한계

- **소규모 벤치마크**: 313개 QA 쌍은 임상 질문의 전체 분포를 대표하기 어렵다. 특정 과(specialty), 언어, 기록 형식에 편향될 수 있다
- **정확도 측정 방식 불명**: 개방형 임상 QA에서 정확도를 어떻게 판정했는지(LLM 자동 채점? 인간 임상의 평가?) `미제공:` 이 기준이 결과 해석에 결정적이다
- **hop 주석 신뢰성**: hop count 주석이 단일 저자에 의해 이루어졌는지, 복수 주석자 간 일치도가 검증됐는지 `미제공:`
- **MedAlign의 대표성**: MedAlign은 미국 내 특정 의료기관 기반 영어 EHR이다. 국내 EMR 환경에서의 재현은 `질문:` 별도 검증 필요
- **절대 정확도의 맥락**: hop=1에서 30.6%라는 수치가 어떤 기준(random baseline이 얼마인지, 다른 비교군이 있는지)에서 낮은 건지 `미제공:`
- **단일 저자 논문**: UCSF 단독 저자(Sanjay Basu). 독립 검증·재현이 중요하다. 주요 발견이 방법론 선택(hop 주석 방식, 정확도 판정 기준)에 민감할 수 있다
- **이해충돌 점검**: `질문:` 저자가 특정 모델 공급사와 연관이 있는지 확인 필요

## 논문 5. Tabular FM Survival: 테이블 파운데이션 모델로 임상 생존 분석

**원문**: https://arxiv.org/abs/2606.12006
**출처**: arXiv (preprint) · AIiH 2026 게재 승인 (동료심사 완료)
**검증 수준**: 후향 (MIMIC-IV ICU, eICU 두 대규모 코호트)

### 임상적 질문

"TabPFN, TabDPT, TabICL 등 범용 테이블 파운데이션 모델이 임상 생존 분석(time-to-event prediction, 주로 ICU 사망 예측)에 직접 적용될 수 있는가, 그리고 생존 분석 전용 헤드(MTLR)를 결합하면 기존 방법을 넘어설 수 있는가." 이 논문이 다루는 임상 문제는 ICU 환자의 사망 위험 예측이다.

임상 미충족 수요와의 접점: `외부 맥락:` ICU 사망률 예측 모델(예: APACHE, SOFA 스코어)은 이미 임상에서 쓰이지만 정확도 한계와 환자 특이성 부족이 지적된다. 기계학습 기반 생존 예측 모델이 임상적으로 유용한 개선을 줄 수 있는지가 핵심 질문이다.

### 검증 수준

**후향 (두 대규모 공개 ICU 코호트).**

코호트:
- **MIMIC-IV** (Medical Information Mart for Intensive Care IV): 미국 Beth Israel Deaconess Medical Centre 기반 단일 기관 ICU 데이터베이스. `외부 맥락:` MIMIC-IV v2.0 기준 71,935건 ICU 입원 포함 (2008~2019)
- **eICU**: 미국 208개 병원 약 200,859건 ICU 입원 기록. `해석:` eICU는 다기관 데이터로 지리적·기관 다양성이 있어 MIMIC-IV 단독보다 일반화 가능성을 높여준다

수치 (원문 보고):
- **MIMIC-IV, TabDPT-FT-MTLR**: C-index 0.856 (DeepSurv 대비 +1.4%, 최적 zero-shot 대비 +6.7%)
- **eICU, TabICL-FT-MTLR**: C-index 0.797 (DeepSurv 대비 +1.7%, 최적 zero-shot 대비 +6.4%)
- 파인튜닝(fine-tuning)이 zero-shot 적용 대비 일관된 이점 확인 (원문 보고)

`해석:` C-index 0.856과 0.797은 각각 단일 기관(MIMIC-IV)과 다기관(eICU) 후향 코호트 내 평가다. DeepSurv 대비 +1.4~1.7%는 통계적으로 유의한지 `미제공:` — 신뢰구간이나 p값 없이는 임상적 유의미성 주장에 주의가 필요하다. AIiH 2026 게재 승인으로 동료심사를 거쳤으나, 후향 단일·양기관 평가라는 검증 수준은 변하지 않는다.

### 임상 적용까지의 거리

중간. 임상 생존 분석 방법론 중 가장 임상 가까이 온 논문이나, 전향 검증이 없다.

1. **현재 기여**: 범용 테이블 FM을 생존 분석에 이식하는 경량 파이프라인(MTLR 헤드 결합)의 타당성 확인. 연구자가 즉시 시도할 수 있는 레시피
2. **남은 단계**:
   - 전향적 임상 코호트(별도 기관, 다른 국가, 다른 시기)에서의 외부 검증
   - C-index 개선이 임상 의사결정(치료 강도 조정, 완화 치료 전환 등)으로 이어져 환자 결과를 실제로 바꾸는지 임상시험 근거 필요
   - `외부 맥락:` MIMIC-IV는 연구자에게 친숙한 벤치마크지만 단일 미국 기관 데이터로, 다른 환경으로의 일반화는 별도 검증 없이 가정할 수 없다
3. **워크플로 통합**: ICU 전자의무기록(EMR)과 실시간 연동, 임상 알림 통합 등 구현 장벽 존재
4. `미제공:` 코드 공개 여부가 초록에 명시되지 않아 재현 가능성 판단 보류

### 수요자 관점

| 수요자 | 의미 |
|--------|------|
| **환자 (ICU)** | 직접 영향 없음. 중기적으로 더 정확한 사망 위험 예측이 ICU 자원 배분·치료 강도 결정에 기여 가능 |
| **중환자의학과 임상의** | DeepSurv 대비 소폭 개선된 생존 예측 도구. 그러나 전향 검증 없이 임상 의사결정에 바로 쓰기 어렵다 |
| **병원 정보 시스템·연구팀** | 공개 테이블 FM(TabPFN 등)을 기존 EHR 데이터로 생존 모델에 활용하는 경량 파이프라인으로 연구 목적 적용 가능 |
| **지불자** | ICU 자원 배분 효율화 도구로서 잠재적 비용 절감 논의가 가능하나, 임상 결과 개선 근거 없이는 급여 논의 불가 |

### 위험·한계

- **단일 기관 원천 한계**: MIMIC-IV는 미국 단일 기관 데이터로 인구통계·진료 패턴·EMR 기록 방식에서 편향이 있다. eICU가 다기관이지만 미국 기관에 한정된다
- **C-index의 한계**: C-index는 예측의 판별력(discrimination)을 측정하지만, 보정(calibration)은 별도 평가가 필요하다. 모델이 위험도를 맞는 방향으로 순위를 매기는 것과 실제 위험 확률을 정확히 예측하는 것은 다르다
- **임상 결과 연결 부재**: C-index 개선이 환자 예후 개선으로 이어지는 증거는 없다. 이 논문은 예측 정확도 연구이며 임상 개입 연구가 아니다
- **통계적 유의성 불명**: +1.4%, +1.7% 개선의 95% 신뢰구간이나 p값 `미제공:` — 소폭 차이가 우연일 가능성 배제 불가
- **테이블 FM의 한계**: 파운데이션 모델의 사전학습이 어떤 도메인 데이터로 이루어졌는지에 따라 임상 데이터 적용 시 분포 이동 문제 발생 가능. `미제공:` 각 FM의 사전학습 데이터 구성 상세 기술 없음
- **과대해석 경계**: "competitive or superior performance" 표현은 선택된 두 ICU 코호트 기준이며, 전향 임상 성능을 보증하지 않는다

**임상 관점 핵심 메시지**: 이번 주 5편 중 임상 현장에 가장 즉각적 함의를 갖는 논문은 **EHR 추론 실패(논문 4)**와 **공정성 xCI(논문 3)**다. 논문 4는 이미 임상 배포가 논의되는 LLM 도구의 구체적 실패 구조를 정량화한다 — hop=1에서도 30.6%라는 낮은 정확도는 임상 배포 설계에서 즉시 고려해야 할 수치다. 논문 3은 이미 쓰이고 있는 CVD 위험 모델에 공정성 감사 도구를 적용하므로 실용 기여가 명확하다. **테이블 FM 생존 분석(논문 5)**는 두 대규모 ICU 코호트에서 후향 검증이 이루어진 점에서 방법론적 기반이 있으나, 전향 검증과 임상 결과 연결 없이는 진료 도입 논의가 이르다. **암 FM 평가(논문 2)**는 임상 배포 전 체계적 평가라는 점에서 중요하나 외부 독립 검증이 없다. **scGTN(논문 1)**은 임상 거리가 가장 멀고 현 단계에서 기초 연구 도구로서 가치를 갖는다.
