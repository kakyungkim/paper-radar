## 1. scGTN: 단일세포 RNA 클러스터링 샴 그래프 트랜스포머

**논문**: "scGTN: Deep Siamese Graph Transformer Network for Single-cell RNA Sequencing Clustering"
**링크**: https://arxiv.org/abs/2606.18672
**상태**: arXiv preprint (미동료심사) · 2026-06-17
**코드/데이터**: 미확인 — 논문 내 저장소 예고가 있으나 공개 GitHub URL 검색 결과에서 확인 불가 (초록 기준)

---

### 문제의식 선행

scRNA-seq 클러스터링 기존 방법들은 데이터의 극도 희소성(sparsity)과 높은 기술 잡음(dropout 현상)을 어느 정도 다루지만, 세포 간 구조 정보(inter-cellular structural dependency)를 명시적으로 통합하지 못했다. 세포 유형 분류는 단일 세포 발현 벡터 수준에서 이루어지는 경우가 많아, 유사한 발현 프로파일을 가진 세포들 사이의 위상 관계가 무시된다.

### 무엇이 새로운가

**저자가 주장하는 새로운 점**:
- 유전자 발현 프로파일과 세포 간 구조 의존성을 **동시에** 통합하는 샴 그래프 트랜스포머 네트워크(Siamese Graph Transformer Network).
- 두 개의 증강 그래프 뷰(augmented graph views)를 구성해 보완적 세포 간 정보를 듀얼 뷰로 포착.
- 그래프 트랜스포머가 **최단 경로 정보(shortest-path information)**와 **노드별 거리(node-wise distances)**를 명시적으로 포함해 세포 간 위상 관계를 풍부하게 인코딩.
- **최적 수송(optimal transport)** 전략으로 자기 지도(self-supervised) 방식의 클러스터링 유도.

**실제로 새로운 점 구분**:
- 외부 맥락: scRNA-seq 그래프 기반 클러스터링은 scGAE, scGAC, scDSC 등에서 이미 시도됐다. 샴 네트워크를 scRNA-seq에 적용한 사례도 scSiameseClu(arXiv 2505.12626, IJCAI 2025)가 선행한다. scSiameseClu 역시 이중 증강 모듈 + 최적 수송 클러스터링을 결합했다.
- 해석: scGTN이 scSiameseClu 대비 차별화되는 요소는 **그래프 트랜스포머에 최단 경로·노드별 거리를 명시 인코딩**하는 부분이다. 이 구조적 편향(structural inductive bias) 도입이 진짜 신규성의 핵심이다. 단, 이 두 논문이 얼마나 독립적으로 설계됐는지, scSiameseClu를 베이스라인으로 비교했는지는 초록 기준으로 확인 불가.
- 검토필요: "기존 방법 대비 일관되게 우수한 성능"이라는 표현은 정성적이며, 어떤 데이터셋에서 얼마나 NMI/ARI가 향상됐는지 구체 수치가 초록에 없다.

### 핵심 가정

1. **KNN 그래프가 세포 간 실제 관계를 포착한다**: 발현 유사도 기반 KNN 그래프가 실제 세포 유형 경계를 반영한다고 전제한다. 추정: scGTN은 PCA 축소 후 코사인 거리 또는 유클리드 거리로 KNN을 구성할 가능성이 높다. 그러나 scRNA-seq의 기술 잡음이 크면 KNN 그래프 자체가 노이즈를 반영해, 최단 경로 정보도 왜곡된다.
2. **최단 경로가 발현 공간의 의미 있는 세포 관계를 인코딩한다**: 그래프 거리(hop count)가 세포 분화 계통 거리와 일치한다고 가정한다. 세포 유형이 연속 스펙트럼을 이룰 때(예: 세포 분화 중간 단계)는 이산적 그래프 거리가 적합하지 않을 수 있다.
3. **두 증강 뷰가 서로 독립적 정보를 담는다**: 샴 구조에서 두 뷰는 다른 노이즈를 가진 동일 세포를 표현해야 한다. 증강 방식이 발현 벡터 마스킹인지, 엣지 드롭아웃인지 초록 기준으로 미확인이며, 두 뷰가 지나치게 유사하면 샴 구조의 이점이 사라진다.
4. **최적 수송으로 클러스터 할당을 유도할 수 있다**: 최적 수송 기반 클러스터링은 클러스터 크기 균형(balanced assignment)을 전제하는 경우가 많다. 실제 scRNA-seq 데이터는 희귀 세포 유형이 극도로 소수여서 이 균형 가정이 깨진다.

**가정이 깨질 때**: 연속 분화 궤적이나 소수 희귀 세포 유형이 포함된 데이터셋에서 그래프 구조 기반 가정들이 연쇄적으로 붕괴할 수 있다. 특히 공간 전사체(spatial transcriptomics) 데이터처럼 세포 위치 정보가 추가적 구조를 제공하는 경우, scGTN의 그래프가 발현 유사도만으로 구성됐다면 위치 정보를 활용하는 방법에 뒤처진다.

### 검증 수준

- in silico 검증: "다수 벤치마크 scRNA-seq 데이터셋"에서 평가 (초록 기준).
- 구체적 데이터셋 이름, NMI/ARI 수치 미기재 (초록 기준).
- 외부 독립 데이터셋 검증: 질문 — 벤치마크 구성과 독립성 불명.
- ablation: 미제공 (초록 기준) — 최단 경로 인코딩 제거, 단일 뷰, 최적 수송 제거 실험 여부 불명.
- 공간 전사체 또는 멀티오믹스 데이터로의 확장 검증: 미제공.

해석: 수치 없는 초록("consistently outperforms")은 실제 성능 개선 폭을 평가하기 어렵다. IJCAI 2025 게재 scSiameseClu(동일 컨셉: 샴 + 최적 수송)가 7개 데이터셋에서 9개 베이스라인을 이긴 것이 이 분야 최신 기준이다. scGTN이 scSiameseClu를 명시적으로 비교했는지가 신규성 판단의 핵심이다.

### 강점 / 한계

**강점**
- 최단 경로·노드별 거리를 그래프 트랜스포머에 통합하는 구조적 설계는 발현 벡터만 보는 방법보다 세포 위상 관계를 명시적으로 반영한다.
- 자기 지도(self-supervised) 방식으로 레이블 없이 동작 — 레이블이 희소한 scRNA-seq 분석 현실에 적합하다.
- 최적 수송은 클러스터 붕괴(representation collapse) 문제를 완화하는 것으로 알려져 있다(외부 맥락).

**한계**
- 코드 미확인으로 재현 검증 불가 (현시점).
- scSiameseClu 등 선행 샴 기반 scRNA-seq 클러스터링 방법과의 직접 비교 여부 초록에 없음.
- 구체 성능 수치 미기재로 정량 비교 불가.
- 희귀 세포 유형에서 최적 수송 균형 가정이 성립하지 않을 수 있다.
- 대규모 데이터셋(수십만 세포)에서 그래프 트랜스포머의 메모리·연산 비용이 bottleneck이 될 가능성 (외부 맥락: 그래프 트랜스포머는 노드 수에 대해 O(n²) 복잡도를 가질 수 있음).

### 내 연구 적용 메모 (전략 노트)

- 전략 노트: scRNA-seq 클러스터링 파이프라인 업그레이드 검토 시, 코드 공개 이후 scSiameseClu(코드 공개, IJCAI 2025)와 직접 비교하는 것이 합리적 진입점이다. scGTN의 최단 경로 인코딩이 실제로 세포 분화 궤적이나 세포 상태 전환을 더 잘 잡는지가 평가 핵심이다.
- 외부 맥락: 멀티오믹스(scATAC-seq + scRNA-seq) 데이터에서 두 오믹스 레이어를 각각 그래프로 구성하고 샴 구조로 통합하는 확장이 이론적으로 가능하다. 저자들이 이 방향을 향후 연구로 언급하는지는 초록 기준 미제공.
- 질문: 논문 본문에서 scSiameseClu와의 비교 실험이 있는지, 구체적 데이터셋(10x Genomics 표준 데이터셋 등)에서의 NMI/ARI 수치가 제공되는지 확인 필요.

## 2. Probing+Fusion Cancer FM: 멀티모달 암 분석 파운데이션 모델 체계적 평가

**논문**: "Probing, Fusion, and Trustworthiness: A Systematic Evaluation of Foundation Model Representations for Multimodal Cancer Analysis"
**링크**: https://arxiv.org/abs/2606.17115
**상태**: arXiv preprint (미동료심사) · 2026-06-15
**코드/데이터**: 공개 안 됨 — 내부 상업 코호트(IH-BC, IH-NSCLC) 사용으로 데이터 미공개. 코드 공개 여부 초록 기준 미확인.

---

### 기존 한계 → 이번 해결

기존 의료 파운데이션 모델 평가는 주로 공개 코호트(TCGA 등)에서 이루어지므로, 실제 상업·임상 환경에서의 분포 이동(distribution shift) 하 일반화를 다루지 못했다. 이 논문은 실세계 상업 코호트에서 WSI(전체 슬라이드 이미지)·전사체 두 모달리티를 함께 평가해 "파운데이션 모델이 실제로 배포될 때 어떻게 작동하는가"를 물었다.

### 무엇이 새로운가

**저자가 주장하는 새로운 점**:
- 단일 평가 프레임워크 안에 세 축 — **probing**(단일 FM 표현 성능), **fusion**(이미지-오믹스 결합 전략), **trustworthiness**(컨포멀 예측 기반 신뢰도) — 을 통합.
- 공개 코호트가 아닌 실세계 상업 코호트(IH-BC: 유방암, IH-NSCLC: 비소세포폐암)로 분포 이동 조건을 명시적으로 검증.
- WSI 파운데이션 모델 5개를 8개 분류 과제에서 동시 비교.
- **컨포멀 예측(conformal prediction)**으로 FM 기반 파이프라인의 예측 커버리지·신뢰도를 수량화.

**실제로 새로운 점 구분**:
- 외부 맥락: WSI FM 벤치마킹 논문(UNI, CONCH, Prov-GigaPath 등 모델 비교)은 이미 다수 존재한다. 그러나 오믹스(전사체)를 명시적으로 포함한 이미지-오믹스 융합 벤치마킹과 컨포멀 예측 신뢰도 평가를 함께 다룬 연구는 상대적으로 드물다.
- 해석: 이 논문의 진짜 기여는 **평가 프레임워크 설계** 자체다. 새 모델을 제안하는 것이 아니라, 기존 FM들이 "실제 배포 조건에서 어떤 상황에서 실패하는가"를 체계적으로 밝히는 것이 가치다.
- 미제공: 5개 FM의 이름, 3가지 융합 전략의 구체적 설계, 8개 분류 과제의 정체, 주요 성능 수치(AUC 등)는 초록 기준 미기재.

### 핵심 가정

1. **상업 코호트가 실제 분포 이동을 대표한다**: IH-BC/IH-NSCLC가 TCGA 대비 다른 분포를 가진다고 전제한다. 그러나 이 코호트가 어느 의료기관에서, 어떤 스캐너로 수집됐는지 초록에 없으며, 이 정보 없이는 "분포 이동"의 구체성을 평가하기 어렵다.
2. **probing(고정 표현 + 선형 헤드)이 FM의 실용적 표현력을 반영한다**: Linear probing은 FM 표현의 충분한 통계로 작동한다고 가정하나, 실제 임상 배포에서는 파인튜닝이 기본이다. probing 결과가 파인튜닝 성능과 얼마나 일치하는지는 별도 검증이 필요하다.
3. **컨포멀 예측이 의미 있는 커버리지를 제공한다**: 컨포멀 예측은 교정 셋(calibration set)이 테스트 셋과 교환 가능(exchangeable)해야 한다. 분포 이동 조건에서 이 가정이 성립하지 않으면 커버리지 보장이 무의미해진다.
4. **이미지와 전사체 표현이 상호 보완적이다**: 두 모달리티가 독립적 예측 신호를 담는다고 가정한다. 연구 발견("이미지와 오믹스는 보완적 신호를 담고 있다")이 이를 지지하지만, 어떤 과제에서 어떤 모달리티가 우세한지는 초록 수준에서 불명확하다.

**가정이 깨질 때**: 컨포멀 예측에서 교환 가능성 가정이 분포 이동 조건에서 무너지면, 보고된 커버리지 보장이 실제 임상 환경에서 성립하지 않는다. 이는 trustworthiness 주장의 근본을 흔든다.

| 데이터 누수 신호 | 질문: probing 헤드 학습에 쓴 레이블과 테스트 레이블 분리 여부 불명 |

### 검증 수준

- in silico 검증: 2개 내부 상업 코호트 × 8개 분류 과제 (초록 기준).
- 공개 코호트(TCGA 등) 검증: 미제공 (초록 기준) — 공개 코호트와의 비교 없이 상업 코호트 결과만으로는 커뮤니티 재현 불가.
- 외부 다기관 검증: 미제공.
- ablation: 미제공 (초록 기준).

해석: 이 논문은 "내 FM이 더 낫다"는 경쟁 논문이 아니라 "현재 FM들이 실제로 어떻게 작동하는가"를 분석하는 평가 논문이다. 따라서 수치보다 **어떤 조건에서 어떤 FM이 실패하는가**라는 패턴이 더 중요하다. 초록만으로는 이 패턴이 드러나지 않는다는 점이 분석 한계다.

### 강점 / 한계

**강점**
- 세 축(probing/fusion/trustworthiness) 통합 프레임워크는 FM 배포 결정 시 체계적 기준을 제공한다.
- 실세계 상업 코호트 사용 — 임상 AI 배포 맥락에 가장 가까운 평가 환경.
- 컨포멀 예측 기반 trustworthiness 평가는 규제 심사(FDA AI/ML 지침 방향)와 정렬된다.
- 멀티모달 융합이 단일 모달 우위일 때와 아닐 때를 구분하는 조건 파악.

**한계**
- 데이터 비공개 — 커뮤니티 재현 불가. 벤치마킹 논문의 가장 큰 한계.
- 코드 미확인 — 평가 파이프라인 재사용 불가 (현시점).
- 5개 FM 이름 미공개 (초록 기준) — 독자가 어떤 모델이 평가됐는지 모르는 상태에서 결론 해석.
- 컨포멀 예측의 분포 이동 조건 적용 타당성 미검증.
- 공개 코호트 병행 검증 부재 — 내부 코호트 결과의 일반화 가능성 불명.

### 내 연구 적용 메모 (전략 노트)

- 전략 노트: WSI + 오믹스 멀티모달 파이프라인 설계 시, 이 논문의 세 가지 융합 전략 비교 결과(본문 확인 필요)가 어떤 전략이 오믹스-이미지 보완성을 가장 잘 활용하는지 판단 근거가 될 수 있다. 코드 공개 시 자체 암 코호트에서 probing 재현 실험 검토.
- 질문: 3가지 융합 전략의 구체적 이름(예: early fusion, late fusion, cross-attention)과 어느 전략이 우세한지 초록에 없음. 논문 본문 확인 필요.
- 외부 맥락: 컨포멀 예측을 임상 AI에 적용하는 논문들이 2025~2026년 급증하고 있다. 커버리지 보장이 분포 이동 하에서도 유지되는지가 현재 분야 핵심 질문 중 하나다.

## 3. Fair Concordance xCI: 그룹 조건부 C-지수로 생존 예측 공정성 평가

**논문**: "Towards Fair Predictions: Group Conditional Concordance Index to Quantify Fairness in Time-to-Event Prognostication"
**링크**: https://arxiv.org/abs/2606.16872
**상태**: arXiv preprint (미동료심사) · 2026-06-15 · AMIA Summit 발표 확인
**코드/데이터**: 미제공/미확인 — 초록 기준 코드 저장소 언급 없음

---

### 메커니즘 핵심 가정

xCI가 작동하는 이유: Harrell의 C-지수는 모든 쌍(pair)에 걸쳐 예측 순위가 맞는 비율을 세는데, 이 집계 과정에서 그룹 간 불균등한 기여가 숨겨진다. xCI는 그룹 멤버십 조건부로 이 계산을 분리해, 그룹 내(within-group)와 그룹 간(cross-group) 랭킹 정확도를 독립적으로 측정한다. 수학적으로 CI는 모든 가능한 그룹 쌍에 걸친 xCI의 가중 평균임을 증명한다(논문 결과, 초록 기준). 이 분해가 가능하려면 그룹 레이블이 관측 가능하고, 중도절단 메커니즘이 그룹에 무관하게(IPCW 가중치로 처리 가능한 방식으로) 작동해야 한다.

### 무엇이 새로운가

**저자가 주장하는 새로운 점**:
- Harrell C-지수를 그룹 조건부로 확장한 **xCI(group-conditional Concordance Index)** 신규 제안.
- 그룹 내와 그룹 간 랭킹 정확도를 별도 측정해 전체 C-지수가 가렸던 집단 간 편향 노출.
- **IPCW(역확률 중도절단 가중치, inverse probability of censoring weights)** 기반 일치 추정량 개발 — 우측 중도절단(right-censored) 데이터에서 편향 없는 추정 가능.
- CI가 xCI들의 가중 평균임을 수학적으로 증명.

**실제로 새로운 점 구분**:
- 외부 맥락: 생존 분석 공정성 연구는 주로 교란 제거나 공정성 정규화 학습에 집중했다. 공정성을 **새 평가 지표**로 제안한 접근은 상대적으로 드물며, 중도절단 데이터에 IPCW를 결합한 그룹 조건부 C-지수는 문헌에서 처음으로 보인다(검토필요: 이전 유사 제안 존재 여부는 본문 관련 연구 섹션 확인 필요).
- 해석: 이 논문의 핵심 기여는 **측정 도구의 제안**이다. 알고리즘이나 모델 구조가 아닌 평가 지표 자체가 새로운 것이다. 지표 제안 논문은 채택 속도가 방법론 논문보다 느리지만, 한번 표준이 되면 영향력이 크다.
- 검토필요: AMIA Summit 발표 (https://amia.secure-platform.com/summit/gallery/rounds/82009/details/14581)가 확인되나, 저널 동료심사는 미완료.

### 핵심 가정

1. **IPCW 가중치가 중도절단 메커니즘을 충분히 교정한다**: IPCW는 중도절단이 관측 공변량(covariate)에만 의존한다는 MAR(Missing at Random) 가정을 전제한다. 중도절단이 그룹 멤버십에 의존하거나 미관측 혼란 변수에 영향받으면 추정량이 편향된다.
2. **그룹 레이블이 의미 있고 관측 가능하다**: 인구통계 그룹(성별, 연령, 인종 등)이 명확히 정의되고 데이터에 있어야 한다. 실제 EHR에서 인종 레이블은 자기 보고(self-reported)이며 불완전하거나 범주 정의가 코호트마다 다를 수 있다.
3. **그룹 쌍 가중치가 공정성 목적과 일치한다**: CI가 xCI들의 가중 평균이 될 때 사용하는 가중치가 어떤 그룹 쌍에 더 높은 가중치를 부여하는지가 공정성 관점에서 중립적이지 않을 수 있다(해석).
4. **순위 기반 지표(C-지수 계열)가 임상적으로 의미 있는 공정성을 포착한다**: C-지수는 개별 예측 위험 점수의 절대적 보정(calibration)이 아닌 순위를 본다. 두 집단에서 순위가 동일하게 정확해도 절대 위험 추정치가 한 집단에서 체계적으로 과대·과소 추정될 수 있다.

**가장 취약한 가정**: MAR 가정 하 IPCW. 심혈관 위험 코호트에서 추적 소실(loss-to-follow-up)이 사회경제적 지위·보험 여부 등 미관측 변수와 연관되면 IPCW가 중도절단 편향을 완전히 제거하지 못한다.

### 재현 가능성

| 항목 | 상태 |
|------|------|
| 코드 | 미제공 (초록 기준) |
| 케이스 스터디 데이터 1 | Framingham Offspring, MESA, ARIC 조화 데이터 — 각 공개/신청 가능 코호트이나 조화 파이프라인 미공개 |
| 케이스 스터디 데이터 2 | Truveta EHR — 상업 데이터베이스, 외부 접근 불가 |
| xCI 추정량 구현 | 미공개 (현시점) — scikit-survival IPCW와 유사한 구조로 추정되나 확인 불가 |
| 베이스라인 | 기존 CVD 위험 예측 모델(Framingham Risk Score 등 추정) |
| 데이터 누수 신호 | 낮음 — 평가 지표 논문으로 학습 없음; 단, Truveta 데이터 접근 재현 불가 |

### 검증 수준

- 케이스 스터디 1: Framingham Offspring + MESA + ARIC 조화 데이터 — 심혈관 위험 예측 공정성 평가 (후향 코호트, 공개 데이터).
- 케이스 스터디 2: Truveta 대규모 EHR — 기존 CVD 위험 예측 모델의 그룹 간 편향 감지 (후향, 데이터 비공개).
- "xCI가 기존 C-지수로 놓친 그룹 간 편향을 식별했다"는 주장: 구체적 수치(어느 그룹에서 얼마나 편향) 초록 기준 미기재.
- 전향 검증: 미제공.
- 시뮬레이션 검증: 미제공 (초록 기준).

해석: 지표 제안 논문으로서 검증 수준은 "이 지표가 기존 지표가 놓친 편향을 실제로 잡아내는가"이다. 두 케이스 스터디에서 편향 감지 성공 사례를 보인 것은 개념 증명(proof-of-concept) 수준이며, 다른 임상 과제나 코호트에서의 일반적 유용성은 추가 검증이 필요하다.

### 강점 / 한계

**강점**
- 개념적 명확성: CI가 xCI의 가중 평균임을 수학적으로 증명한 것은 지표 간 관계를 투명하게 만든다.
- IPCW를 활용해 중도절단 문제를 통계적으로 처리 — 기존 공정성 지표(분류기 공정성 지표)의 생존 분석 직접 적용 문제를 우회.
- 두 개 대규모 코호트(Framingham/MESA/ARIC, Truveta)에서 실제 편향 감지 사례 제시.
- 해석 단순성: 의미가 직관적이어서 임상의·규제 기관과 소통 가능.

**한계**
- 코드 미공개 — 즉시 채택 장벽.
- Truveta 데이터 비공개 — 두 케이스 스터디 중 규모가 더 큰 것을 재현 불가.
- 순위 기반 공정성만 측정 — 보정(calibration) 공정성은 별도 지표 필요.
- IPCW의 MAR 가정이 실제 EHR에서 성립하는지 검증 없음.
- 다그룹(>2) 동시 비교 시 xCI 해석 복잡도 증가 및 다중 비교 문제 미언급.

### 내 연구 적용 메모 (전략 노트)

- 전략 노트: 임상 생존 예측 모델(예: 암 예후, 심혈관 위험)을 개발하거나 평가할 때 xCI를 보조 지표로 추가하는 것이 규제 및 의료 형평성 관점에서 방어적 선택이다. 코드 공개 시 scikit-survival 기반 래퍼로 구현해 기존 파이프라인에 통합 검토.
- 외부 맥락: FDA의 AI/ML 기반 SaMD(의료기기 소프트웨어) 심사에서 서브그룹 성능 검증을 요구하는 방향이 강화되고 있다. xCI는 이 서브그룹 성능을 생존 분석 맥락에서 한 수치로 표현하는 도구가 될 수 있다.
- 질문: xCI 코드를 저자들이 Python 패키지로 공개할 계획인지, AMIA 발표 자료가 공개됐는지 확인 필요.

## 4. Clinical EHR Reasoning Failure: 임상 EHR QA에서 트랜스포머 합성 추론 한계

**논문**: "Compositional Reasoning Depth Predicts Clinical AI Failure: Empirical Evidence Consistent with Transformer Compositionality Limits in Electronic Health Record Question Answering"
**링크**: https://arxiv.org/abs/2606.16890
**상태**: arXiv preprint (미동료심사) · 2026-06-15
**코드/데이터**: 미제공 — 코드 및 주석 데이터셋 공개 여부 초록 기준 미확인

---

### 문제의식 선행

대형 언어 모델(LLM)이 임상 EHR 질의응답에서 실패하는 원인이 불분명했다. 단순히 "의료 지식 부족"인가, 아니면 구조적 추론 능력의 한계인가. 이 논문은 실패 원인을 추론 단계 수(hop count)라는 조작 가능한 변수로 정량화하려 한다.

### 무엇이 새로운가

**저자가 주장하는 새로운 점**:
- **hop-count 분류 체계(taxonomy)**: 임상 EHR 질문을 답하기 위해 필요한 추론 단계 수로 질문을 1~4 hop으로 분류.
- 313개 MedAlign EHR QA 쌍에 hop 레이블 부착 — 이 주석 데이터셋 자체가 공헌.
- Claude Sonnet(claude-sonnet-4-6), GPT-4o, GPT-5(gpt-5.4-2026-03-05) 세 모델에서 hop 수 증가에 따른 단조 정확도 감소를 재현.
- "트랜스포머 합성 추론 한계(transformer compositionality limits)"라는 이론적 틀로 임상 AI 실패를 설명.

**실제로 새로운 점 구분**:
- 외부 맥락: 트랜스포머가 multi-hop 추론에 취약하다는 것은 일반 NLP(HotpotQA, 2WikiMultiHopQA 등)에서 알려진 현상이다. 이를 임상 EHR에 명시적으로 적용해 hop 분류 체계를 설계하고 세 최신 상용 모델에서 재현한 것이 이 논문의 기여다.
- 해석: 진짜 신규성은 **임상 EHR에 특화된 hop 분류 체계**와 **세 모델 간 일관된 재현**이다. 이론적 기반(트랜스포머 합성 추론 한계)은 기존 문헌에서 가져온 것이고, 새 이론을 만든 것이 아니다.
- 검토필요: 논문 내 "claude-sonnet-4-6"이 이 보고서를 작성한 모델과 동일한 모델이다. 자기 평가(self-evaluation) 상황은 아니지만, 동일 모델이 피험 대상인 논문을 분석한다는 점을 명시한다.

### 핵심 가정

1. **hop count가 추론 복잡도를 충분히 포착한다**: 추론 단계 수를 1~4로 세는 것은 질문 구조를 단순화한다. 같은 hop=3인 질문도 각 단계의 의미 복잡도가 크게 다를 수 있다. Hop이 같아도 EHR 내 관련 정보의 분산도(얼마나 많은 노트에 흩어져 있나)에 따라 난이도가 달라진다.
2. **MedAlign이 실제 임상 EHR 질의를 대표한다**: MedAlign은 313개 질문이 임상 의사가 생성한 것이라고 알려져 있으나, 특정 의료기관·전공·진료 흐름을 반영한다. 다른 의료기관 EHR 환경에서의 일반화는 별도 검증 필요.
3. **정확도(binary correct/incorrect)가 임상 AI 실패를 적절히 측정한다**: 실제 임상에서는 "완전 오답"보다 "부분적으로 맞지만 위험한 답"이 더 문제다. 이진 정확도는 이 뉘앙스를 잡지 못한다.
4. **zero-shot 조건이 임상 AI 배포 조건을 대표한다**: 실제 임상 AI 도구는 시스템 프롬프트, RAG(검색 증강), 구조화된 EHR 추출 파이프라인과 함께 사용된다. Zero-shot 단독 평가는 실제 배포보다 훨씬 제약된 조건이다.

**가장 취약한 가정**: Zero-shot 조건에서의 성능 저하가 실제 임상 배포 조건(RAG + 시스템 프롬프트 + few-shot)에서도 동일하게 나타나는지 미검증. Claude Sonnet의 extended thinking 조건과 zero-shot 비교가 논문 내에 있으나(within-model ablation), extended thinking이 hop=4에서 얼마나 개선됐는지 초록 기준 수치 없음.

### 재현 가능성

| 항목 | 상태 |
|------|------|
| 코드 | 미제공 (초록 기준) |
| 주석 데이터셋 | 313개 MedAlign QA + hop 레이블 — MedAlign 원 데이터는 공개이나 hop 주석 공개 여부 미확인 |
| 평가 모델들 | claude-sonnet-4-6, gpt-4o, gpt-5.4-2026-03-05 — 상용 API 접근 필요 (재현 가능) |
| 하이퍼파라미터 | 프롬프트 설계, temperature 등 미제공 (초록 기준) |
| 베이스라인 공정성 | Zero-shot vs. extended thinking(Claude만) — 간접적 ablation |
| 데이터 누수 신호 | 낮음 — 평가 연구이므로 학습 없음; 단, GPT-5 학습 데이터에 MedAlign 포함 가능성 배제 불가 |

### 검증 수준

- in silico 평가: 313개 MedAlign QA, 3개 모델, hop 1~4 (초록 기준).
- 재현 독립성: 세 모델 모두에서 패턴 재현 — "두 제공자(Anthropic, OpenAI), 두 세대(GPT-4, GPT-5)에서 동일 패턴"이 핵심 강점.
- 외부 EHR 데이터셋(non-MedAlign) 검증: 미제공 (초록 기준).
- 임상 결과(환자 안전 사고 등)와 연관 검증: 미제공.
- ablation: 부분적 — zero-shot vs. extended thinking (Claude Sonnet만).

해석: hop=1에서 30.6%, hop=4에서 17.6%로의 단조 감소(원문 기재 수치)는 절대적 정확도 자체가 낮다는 점도 주목해야 한다. hop=1에서도 30.6%라는 것은, 가장 단순한 단일 추론 질문에서도 LLM 3개 중 7할이 틀린다는 의미다. 이는 hop 효과만큼이나 "임상 EHR QA 과제 자체의 어려움" 또는 "평가 방법론의 엄격함"에 대한 논의가 필요함을 시사한다.

### 강점 / 한계

**강점**
- 세 모델에서의 일관된 패턴 재현은 우연이나 모델 특이성을 넘어선 체계적 결과임을 지지한다.
- 임상 EHR hop 분류 체계는 향후 임상 AI 안전성 벤치마크 설계의 기준이 될 수 있다.
- 단일 저자(UCSF), 공개 데이터(MedAlign 기반)로 수행 — 이해충돌 가능성 낮음.
- 결과가 "어떻게 고칠 수 있나"로 직결: hop 수를 줄이는 EHR 구조화, chain-of-thought, RAG 설계.

**한계**
- 코드·주석 데이터 미공개 — hop 분류 체계의 간주관적 신뢰도(inter-rater reliability) 확인 불가.
- 313개 샘플로 hop 1~4 분석 — 각 hop 레벨 샘플 수가 적을 수 있어 통계적 안정성 우려.
- Zero-shot 조건만 평가 — 실제 임상 배포 조건(RAG, CoT, few-shot)에서의 hop 효과 미검증.
- 정확도 평가의 이진화 — 임상 안전 관점에서 중요한 "부분 정확" 또는 "위험한 오답" 구분 없음.
- GPT-5 학습 데이터에 MedAlign 일부 포함 가능성 — 데이터 오염 검증 없음(초록 기준).

### 내 연구 적용 메모 (전략 노트)

- 전략 노트: 임상 EHR 기반 LLM 파이프라인 설계 시, hop 수를 줄이는 방향(복잡 질문을 sub-question으로 분해하는 전처리, 또는 RAG로 관련 컨텍스트를 미리 수집)이 성능 향상의 실용적 경로다. hop 분류 체계는 내부 임상 AI 벤치마크에 즉시 적용 가능한 평가 설계 원칙이다.
- 외부 맥락: extended thinking(chain-of-thought 강화) 조건에서 hop 효과가 얼마나 완화되는지가 현재 이 분야의 핵심 질문이다. 논문 내 within-model ablation 결과가 이를 일부 다루나 초록에서 수치 미공개.
- 질문: hop 주석 데이터셋의 코드 또는 CSV 공개 계획, inter-rater reliability(복수 임상의가 동의한 hop 분류인지) 여부를 본문에서 확인 필요.

## 5. Tabular FM Survival: 테이블 파운데이션 모델 임상 생존 분석 적응

**논문**: "Tabular Foundation Models for Clinical Survival Analysis via Survival-Aware Adaptation"
**링크**: https://arxiv.org/abs/2606.12006
**상태**: arXiv preprint (미동료심사) · 2026-06-10 · AIiH 2026 게재 승인 (동료심사 완료)
**코드/데이터**: 미제공 — 코드 저장소 미공개 (초록 기준). 기반 모델(TabDPT 등)은 별도 공개 저장소 존재.

---

### 기존 한계 → 이번 해결

범용 테이블 파운데이션 모델(TabPFN, TabDPT, TabICL)은 분류와 회귀 과제에서 강력한 in-context learning 능력을 보이지만, **우측 중도절단(right-censored) 생존 데이터**를 직접 처리하는 구조를 갖추지 못했다. 생존 분석 특화 출력 헤드 없이 zero-shot으로 적용하면 중도절단 구조를 무시한다. 이 논문은 MTLR(다중과제 로지스틱 회귀) 헤드를 기존 FM 표현 위에 결합해 경량 파인튜닝으로 이 격차를 메운다.

### 무엇이 새로운가

**저자가 주장하는 새로운 점**:
- 범용 테이블 FM(TabPFN, TabDPT, TabICL) 표현 위에 **MTLR 헤드**를 부착해 임상 생존 분석에 적응.
- Zero-shot 적용(기반 FM 그대로)과 파인튜닝(FT) 조건을 모두 평가.
- MIMIC-IV와 eICU 두 대규모 ICU 코호트에서 검증.

**실제로 새로운 점 구분**:
- 외부 맥락: 같은 방향의 선행 논문이 같은 arXiv 기간 내에 존재한다 — arXiv 2601.22259 "Tabular Foundation Models Can Do Survival Analysis"가 MTLR+TabFM 접근을 먼저 제안했다. 이 논문과의 명시적 비교가 있는지, 기여 차별점이 무엇인지 초록 기준 불명확하다.
- 해석: 이 논문의 기여는 **임상 ICU 코호트(MIMIC-IV, eICU)에서의 대규모 검증**과 여러 FM 아키텍처(TabPFN/TabDPT/TabICL) 비교라고 볼 수 있다. 방법 자체보다 검증의 규모와 다양성이 강점이다.
- 검토필요: arXiv 2601.22259와의 관계 — 동일한 MTLR+TabFM 아이디어를 다른 그룹이 독립적으로 제안한 것인지, 아니면 이 논문이 2601.22259를 확장한 것인지 본문 확인 필요.

### 핵심 가정

1. **테이블 FM의 사전학습 표현이 임상 ICU 데이터로 전이된다**: TabDPT 등은 다양한 테이블 데이터셋으로 학습됐지만, 임상 EHR 데이터(결측치 패턴, 범주형 변수 코딩, 시계열 집계 방식)는 이 사전학습 분포와 다를 수 있다.
2. **MTLR 헤드가 FM 표현의 생존 정보를 충분히 끌어낸다**: MTLR은 시간 구간별 이산 생존 확률을 예측하는 방법이다. FM 표현에서 시간 의존성을 MTLR이 선형적으로 포착한다고 가정한다. 비선형 시간 의존 효과는 놓칠 수 있다.
3. **MIMIC-IV와 eICU의 성능이 일반 임상 코호트로 전이된다**: 두 코호트는 모두 ICU 환자이며 중증도가 높다. 외래·일반 병동 코호트나 특정 암종 생존 분석에서의 성능은 별도 검증 필요.
4. **파인튜닝 데이터 규모가 충분하다**: 작은 임상 코호트(수백~수천 명)에서 파인튜닝이 과적합 없이 수렴한다고 가정한다. MIMIC-IV는 수십만 입원 기록이 있어 괜찮지만, 소규모 기관 코호트로 전이 시 파인튜닝이 오히려 해가 될 수 있다.

**가장 취약한 가정**: 사전학습 분포 전이. TabDPT가 임상 EHR 특유의 결측·코딩 패턴에 얼마나 잘 적응하는지가 실제 임상 배포의 핵심 불확실성이다. DeepSurv 대비 +1.4% 향상이 이 전이 비용보다 큰지는 소규모 기관 코호트에서 재현해야 안다.

### 재현 가능성

| 항목 | 상태 |
|------|------|
| 코드 | 미제공 (초록 기준) |
| 기반 FM 코드 | TabDPT(layer6ai-labs/TabDPT), TabPFN(공개) — 기반 모델은 재현 가능 |
| 학습 데이터 | MIMIC-IV, eICU — PHYSIONET 신청으로 접근 가능 (공개 프로세스) |
| 하이퍼파라미터 | 미제공 (초록 기준) |
| 베이스라인 | DeepSurv(C-index 0.844) 및 zero-shot 모델(C-index 0.802) — 초록 기재 수치 |
| 데이터 누수 신호 | 주의: ICU 코호트에서 시계열 집계 방법에 따라 미래 정보가 특징에 포함될 수 있음(누수 위험). 특징 엔지니어링 세부 사항 미제공 |

### 검증 수준

- in silico 검증: MIMIC-IV (C-index 0.856, TabDPT-FT-MTLR), eICU (C-index 0.797, TabICL-FT-MTLR) — 초록 기재 수치.
- 다양한 생존 벤치마크 추가 검증 포함 (초록 기준, 구체 목록 미기재).
- 외부 다기관 검증: 미제공 — MIMIC-IV와 eICU가 사실상 가장 큰 공개 ICU 코호트이지만, 이 두 코호트가 서로 독립적인지 주의 (eICU는 Philips 시스템 병원들, MIMIC은 BIDMC).
- ablation: 미제공 (초록 기준) — MTLR 헤드 vs. 다른 헤드(Cox, Weibull 등) 비교, FM 표현 고정 vs. end-to-end 미보고.
- AIiH 2026 게재 승인: 동료심사 통과 — 같은 분야 유사 preprint 대비 검증 수준 높음.

해석: TabDPT-FT-MTLR의 C-index 0.856이 DeepSurv 0.844 대비 +1.4%는 통계적으로 유의한지 신뢰 구간 미기재로 판단 불가. 임상적으로 C-index 1% 차이의 의미는 코호트 구성과 사용 목적에 따라 크게 달라진다. Zero-shot 대비 +6.7%(0.802→0.856)는 파인튜닝의 가치를 더 명확하게 보여준다.

### 강점 / 한계

**강점**
- AIiH 2026 동료심사 통과 — 같은 주제 preprint 중 검증 수준이 가장 높다.
- 두 독립적 ICU 코호트(MIMIC-IV, eICU) 검증 — 단일 코호트 과적합 위험 낮음.
- 경량 적응(MTLR 헤드만 학습) 레시피 — 계산 비용 낮고 구현 단순.
- Zero-shot vs. 파인튜닝 비교로 FM 표현의 out-of-box 유용성을 명확히 측정.
- 세 FM 아키텍처(TabPFN/TabDPT/TabICL) 비교로 FM 선택 가이드 제공.

**한계**
- 코드 미공개 — MTLR 헤드 연결 구현 세부 사항 불명.
- ICU 코호트만 검증 — 외래, 암 생존, 일반 병동 코호트 이전 가능성 미확인.
- DeepSurv 대비 수치 개선의 통계적 유의성 미보고.
- 선행 논문(arXiv 2601.22259)과의 관계·차별점 불명확.
- 데이터 전처리(특징 엔지니어링) 세부 미공개 — 누수 위험 확인 불가.

### 내 연구 적용 메모 (전략 노트)

- 전략 노트: 임상 생존 분석 파이프라인 구축 시 TabDPT + MTLR 헤드 조합이 경량 강력 베이스라인으로 시도할 가치가 있다. 기반 모델(TabDPT)은 공개 저장소가 있으므로, 코드 미공개인 MTLR 헤드 부분만 자체 구현하면 재현 실험이 가능하다.
- 외부 맥락: MTLR 헤드는 원래 Fotso & Zitzelberger(2018)가 제안한 방법으로 라이브러리(pycox, torchsurv 등)에 구현이 있다. FM 표현 + MTLR 조합을 만드는 것은 코드 몇 줄 수준의 작업이다.
- 질문: 세 FM 중 어느 것이 어떤 코호트 유형에서 유리한지(TabPFN vs. TabDPT vs. TabICL 비교 패턴), MTLR 외 다른 헤드(DeepHit, CoxPH) 비교가 본문에 있는지 확인 필요.

## 교차 주제 메모

### 1. 코드 공개율 하락 — 실질 재현 가능성 우려

W25 5편 중 코드가 확인된 논문은 0편이다(W24의 2편 대비 하락). Tabular FM Survival과 EHR Reasoning 논문은 기반 데이터·모델이 공개 접근 가능하지만, 실험 코드가 없어 정확한 재현은 어렵다. 코드 공개가 없는 주에는 "방법 아이디어 채택"과 "직접 재현" 가능성을 분리해 판단해야 한다.

### 2. 평가 지표·평가 논문의 부상

W25 5편 중 scGTN만 새 알고리즘을 직접 제안한다. 나머지 4편은 평가 체계(FM 벤치마킹, 공정성 지표, 추론 실패 분석, 적응 전략 비교)를 주 기여로 한다. 이 추세는 분야가 "더 좋은 모델"에서 "모델을 제대로 평가하는 방법"으로 무게중심을 이동하고 있음을 시사한다. 방법론 연구자에게는 좋은 평가 프레임워크 설계가 독립적 연구 기여로 인정받는 환경이다.

### 3. 임상 AI의 구조적 실패 패턴

EHR Reasoning Failure 논문(hop count → 정확도 감소)과 Probing+Fusion Cancer FM 논문(분포 이동 하 FM 일반화)은 서로 다른 각도에서 임상 AI의 체계적 실패 패턴을 분석한다. 두 논문 모두 "얼마나 좋은가"가 아닌 "언제, 왜 실패하는가"를 묻는다. 이 실패 조건 파악이 임상 AI 배포 결정의 핵심 사전 작업이라는 인식이 공유되고 있다.

### 4. 생존 분석과 공정성의 교차

xCI(공정 C-지수)와 Tabular FM Survival은 생존 분석이라는 공통 기반 위에서 서로 다른 문제를 다룬다 — 하나는 공정성 측정, 다른 하나는 예측 성능 개선. 두 접근을 결합하면 "어떤 그룹에서 FM 기반 생존 예측이 공정하지 않은가"라는 자연스러운 후속 질문이 나온다. W25 논문들의 접점 연구로 주목할 조합이다.

### 5. scRNA-seq 클러스터링: 샴 아키텍처 경쟁 구도

scGTN(W25)과 scSiameseClu(IJCAI 2025, arXiv 2505.12626)는 거의 동일한 설계 요소(샴 + 이중 증강 + 최적 수송)를 공유한다. 이 두 논문의 성능을 동일 벤치마크에서 비교하는 것이 단일세포 클러스터링 방법 선택에서 합리적 출발점이다. 코드 공개 여부도 scSiameseClu가 우위(IJCAI 2025, 공개 코드 확인됨).
